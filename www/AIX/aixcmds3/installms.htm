installms 命令

用途

installms ― 安装群集系统管理（CSM）管理服务器软件。

语法

installms [-h]

installms [-f] [-v | -V] {-x | -p pkg_path:[pkg_path...]} [Attr=value ...]

描述

installms 命令安装 CSM 管理服务器软件。 它自动安装 CMS 和它的必备软件。在运行 installms 命令前，必须安装操作系统的支持版本。

这个命令执行以下任务：

  • 创建 CMS 需要的 /csminstall 目录结构。
    注:
    如果 /csminstall 目录不存在，它将由 installms 命令创建。然而，建议在运行 installms 命令之前，在一个独立的文件系统中创建 /csminstall 目录。请参阅 CSM for Linux: Planning and Installation Guide 获取有关 /csminstall 的要求的更多信息。
  • 也可以将所需的 文件集复制到当前 /csminstall 子目录。
  • 将在群集的节点上安装 CSM 所需求的 CSM 脚本和文件复制到正确的 /csminstall 子目录。
  • 设置全局群集系统属性值。

如果使用了 -p 标志，则 installms 命令将需求的文件集从指定的位置复制到 /csminstall 的正确子目录。如果使用了-x 标志，则 installms 命令假定文件已经复制。当 installms 命令安装管理服务器软件时，它使用已经复制到 /csminstall 子目录的文件集。

标志

-c
强制命令复制 pkg_path 中的全部必需文件。通常情况下，installms 将仅复制与 /csminstall 中的文件相同或更新的文件，或 /csminstall 中没有的文件。-c 标志强制 installms 复制所有文件，而不考虑其版本。
-f
忽略现有安装状态强制安装。
-h
把该命令的用法语句写到标准输出。
-p pkg_path
指定一个或多个冒号隔开的安装软件包所在的目录。该标志不能与 -x 一起使用。

如果指定了此选项，命令在找到 ISO CD 映像时将对其进行处理。此命令将按照从目录复制的方式复制这些文件，并一次加载一个映像。CD 映像无需按顺序操作。此命令还将处理 CSM CD 的 ISO 映像，如：

installms -p /tmp/image1:/tmp/image2:/tmp/image3 -n node1
-v | -V
把命令的冗长消息写到标准输出。
-x
指定不复制软件包，并使用 /csminstall 子目录下软件包。该标志不能与 -p 一起使用。

参数

Attr=value...
指定一个或多个由空格分隔的 Attr=value 对。Attr=value 对必须在命令行的最后进行指定。有效属性为：
RemoteShell=full_path_name_of_shell
如果 DSH_REMOTE_CMD 变量不被设置，RemoteShell 属性值为 dsh 命令决定缺省的远程 shell。它还决定在安装期间在节点上设置哪个 shell。管理服务器的缺省属性值为 /usr/bin/rsh。
SetupRemoteShell={yes | no}
SetupRemoteShell 属性值控制在管理服务器和节点上是否设置远程 shell。如果它被设为 yes 或 1，那么设置远程 shell。如果这个属性被设为 no 或 0，不设置远程 shell。这个属性的缺省值是 1。

安全性

该命令要求至群集管理服务器的 root 用户访问权。

退出状态

0
这个命令已成功运行。
1
有一个错误发生。

环境变量

DSH_FANOUT
设置并发远程 shell 命令的最大数。也可以通过 -f 标志设置。
DSH_LIST
指定一个文件，该文件包含构成节点列表的节点集的定义。
DSH_LOG
设置为日志文件的绝对路径和文件名后启用日志记录。此变量将由 --log 和 --noLogging 标志覆盖。
DSH_PATH
设置远程节点上所用的路径。如果未设置 DSH_PATH，将使用远程 shell 的缺省路径。例如，DSH_PATH=$PATH 将远程节点上的路径设置为源节点上所用的相同路径。
DSH_REMOTE_CMD
指定要使用的可执行远程 shell 的路径，而不是缺省路径。
DSH_REMOTE_OPTS
包含将此命令转发到远程节点时在远程命令中指定的选项。
DSH_REPORTS_DIRECTORY
设置为报告目录路径后启用报告和输出文件。此变量将由 --reports 和 --noReporting 标志覆盖。
DSH_TIMEOUT
指定等待来自远程 shell 的响应的时间（以秒为单位）。也可以使用 -t 标志设置。
WCOLL
等同于 DSH_LIST。

标准错误

使用 dshbak 过滤器并生成了标准错误消息时，有关标准错误的所有错误消息都将在所有标准输出消息之前显示。无论是否使用 -c 标志，这都成立。

示例

  1. 要用被事先拷贝到 /tmp/csm 目录下的 CSM 文件集合安装 CMS 管理服务器软件,请输入： 
    installms -p /tmp/csm
  2. 要在管理服务器上用强制选项安装在包含在当前目录下的 CSM 和软件包，请输入： 
    installms -f -p .
  3. 要用已被事先拷贝到 /csminstall/AIX/csm/1.3.0/packages 目录下的 AXI CSM V1.3.0 的软件包来在管理服务器上安装 CSM，请输入： 
    installms -x

定位

/opt/csm/bin/installms
