mkcfsmnt 命令

用途

安装一个 CacheFS 目录。

语法

mkcfsmnt -d PathName -t { nfs | cdrom} ] [ -h RemoteHost ] [ -p { RemoteDirectory | LocalDeviceName } ] [ -c CacheDirectory ] [ -o MountOptions ] [ -b BackingFileSystem ] [ -I | -B | -N ]

描述

mkcfsmnt 命令构造一个将被附加到 /etc/filesystems 文件的项，因而构造一个可用的文件系统用作高速缓存文件系统。 如果是永久安装，则保留该项。如果是临时安装，则为 mount 命令直接使用标志。CacheFS 文件系统用来为访问后备文件系统提供高速缓存。后备文件系统通常是 NFS 安装。

标志

-d PathName 为高速缓存目录指定安装点。
-t 选择要进行高速缓存操作的文件系统。
nfs
指定由 NFS 安装支持 CacheFS 文件系统。
cdrom
指定由 CDROM 文件系统支持 CacheFS 文件系统。（目前不支持。）
-h RemoteHost 指定导出目录的 NFS 服务器。
-p RemoteDirectory 指定安装在指定的路径名上的目录。这通常是将要通过 NFS 安装的远程文件系统或是在 CDROM 情况下的本地设备名（目前不支持。）
-c CacheDirectory 指定 CacheFS 文件系统的位置。这必须是以前通过执行 cfsadmin 命令已创建。
-d RemoteDirectory 指定在指定的路径名上安装的目录。
-o MountOptions 指定依赖于备份文件系统类型的由逗号分隔开的安装选项字符串。例如，如果是 NFS，则这些选项就是典型地通过 -o 选项字符串指定安装的那些选项。要获取可接受的值，请参阅 mount 命令文档。
-b BackingFileSystem 如果已安装，则指定一个后备文件系统。如果未指定，那么命令将在一个临时安装点自行安装。如果未指定这一点，那么必须指定 RemoteHost 和 RemoteDirectory。
-I 将一个项添加到 /etc/filesystems 文件。不安装这个目录。
-B 将一个项添加到 /etc/filesystems 文件且试图安装该文件系统。这个标志是缺省值。
-N 带有指定的选项安装目录，但并不修改 /etc/filesystems 文件。

示例

要指定一个 CacheFS 安装，请输入：

/usr/sbin/mkcfsmnt -t nfs -d /usr/share/man -p /usr/share/man -h host1 -c /cache/cache1 -o ro, intr -N

在本示例中，mkcfsmnt 命令为驻留在本地 /usr/share/man 上的 host1 上的远程目录 /usr/share/man 进行高速缓存。 高速缓存保存在由 cfsadmin 命令创建的 /cache/cache1 中。 因为没有指定 -b 标志，所以 CacheFS 执行 NFS 后备安装。

/usr/sbin/mkcfsmnt -t nfs -d /usr/share/man -p /usr/share/man -h host1 -c /cache/cache1 -b /backs/man -o ro, intr -N

在本示例中，mkcfsmnt 命令对远程目录 /usr/share/man 进行高速缓存，它驻留在本地 /usr/share/man 目录上的 host1 上。高速缓存保存在由 cfsadmin 命令创建的 /cache/cache1 中。后备文件系统已经安装在 /backs/man 上。

文件

/etc/filesystems 列出在系统重新启动过程中要安装的远程文件系统。
