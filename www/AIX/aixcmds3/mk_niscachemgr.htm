mk_niscachemgr 命令

用途

为 nis_cachemgr 守护进程在 /etc/rc.nfs 文件中删除项的注释，并且通过使用 startsrc 命令调用守护进程。

语法

/usr/sbin/mk_niscachemgr [  -I ] | [  -B ] | [  -N ]

描述

mk_niscachemgr 命令为 nis_cachemgr 守护进程在 /etc/rc.nfs 文件中删除项的注释。 mk_niscachemgr 命令通过使用 startsrc 命令启动守护进程。

注：mk_nisd、mk_cachemgr、mk_nispasswdd、rm_nisd、rm_cachemgr 和 rm_nispasswdd 命令做两件事情：
  • 改变 /etc/rc.nfs 中的守护进程启动调用的项。
  • 改变守护进程 src 实体的缺省行为。

例如，如果假设以 -Y 标志启动 rpc.nisd，那么这将不会明确地在 /etc/rc.nfs 项中设置为启动 rpc.nisd 守护进程。相反，执行 chssys 来在启动过程中放置已添加（如果有的话）到守护进程的缺省选项。要验证这些选项是否存在，使用 lssrc -S -s 子系统命令来显示缺省选项。

标志

-I 在 /etc/rc.nfs 文件中删除项的注释以在下一次系统重新启动时启动 nis_cachemgr 守护进程。
-B 在 /etc/rc.nfs 文件中删除项的注释以启动 nis_cachemgr 守护进程，并且使用 startsrc 命令来启动 nis_cachemgr 守护进程。这个标志是缺省值。
-N 使用 startsrc 命令来启动 nis_cachemgr 守护进程。此标志不更改 /etc/rc.nfs 文件。

 

示例

要修改 /etc/rc.nfs 文件以在下一次系统重新启动时调用 nis_cachemgr 守护进程，请输入： 

mk_niscachemgr -I

文件

/etc/rc.nfs 包含 NFS 和 NIS 守护进程的启动脚本。
