ipsectrcbuf 命令

用途

列出 IP 安全子系统中跟踪缓冲区的内容。

语法

ipsectrcbuf [-l {0|1|2}]

描述

如果出现问题的话，IP 安全子系统会维护内存驻留跟踪缓冲区来帮助调试。缓冲区的内容（即最新跟踪消息的修订数）将会放在系统转储中，或者可以通过运行不带参数的该命令来列出。

标志

-l 设置 IP 安全的跟踪级别。默认情况下，九个 IP 安全跟踪中，只有 IPSEC_ERROR 跟踪消息放在缓冲区中。为了启用或者禁用其它的跟踪挂钩，使用带以下值之一的 -l 标志：
0
只有 IPSEC_ERROR 跟踪消息可以写到缓冲区中。这是缺省情况。
1
IPSEC_FILTER、IPSEC_CAPSUL、IPSEC_CRYPTO、
IPSEC_TUNNEL 和 IPSEC_ERROR 跟踪消息
都可以写到缓冲区中去。
2
所有的 IP 安全跟踪消息都放在缓冲区中（它包括
IPSEC_FILTER_INFO、IPSEC_CAPSUL_INFO、
IPSEC_CRYPTO_INFO 和 IPSEC_TUNNEL_INFO
就和那些在级别 1 中的一样）。
