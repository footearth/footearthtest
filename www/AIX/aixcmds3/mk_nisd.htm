mk_nisd 命令

用途

为 rpc.nisd 守护进程在 /etc/rc.nfs 文件中删除项的注释，并且通过使用 startsrc 命令调用守护进程。

语法

/usr/sbin/mk_nisd [  -I ] | [  -B ] | [  -N ] [  -s ] [  -y ] [  -b ]

描述

mk_nisd 命令为 rpc.nisd 守护进程在 /etc/rc.nfs 文件中删除项的注释。 mk_nisd 命令通过使用 startsrc 命令启动守护进程。

注：mk_nisd、mk_cachemgr、mk_nispasswdd、rm_nisd、rm_cachemgr 和 rm_nispasswdd 命令做两件事情：
  • 改变 /etc/rc.nfs 中的守护进程启动调用的项。
  • 改变守护进程 src 实体的缺省行为。

例如，如果假设以 -Y 标志启动 rpc.nisd，那么这将不会明确地在 /etc/rc.nfs 项中设置为启动 rpc.nisd 守护进程。 相反，执行 chssys 来在启动过程中放置已添加（如果有的话）到守护进程的缺省选项。要验证这些选项是否存在，使用 lssrc -S -s 子系统命令来显示缺省选项。

标志

-I 在 /etc/rc.nfs 文件中删除项的注释以在下一次系统重新启动时启动 rpc.nisd 守护进程。
-B 在 /etc/rc.nfs 文件中删除项的注释以启动 rpc.nisd 守护进程，并且使用 startsrc 命令来启动 rpc.nisd 守护进程。这个标志是缺省值。
-N 使用 startsrc 命令来启动 rpc.nisd 守护进程。这个标志不更改 /etc/rc.nfs 文件。
-s 不带 DES 认证来启动 rpc.nisd。如果不使用这个标志，那么 rpc.nisd 的缺省行为是始终带有 DES 认证启动。-s 选项用来使 rpc.nisd 和 NIS（YP）客户机兼容。
-y 使 rpc.nisd 守护进程模拟一个 NIS（YP）服务。这并不是 rpc.nisd 或 mk_nisd 的缺省设置。
-b 使 rpc.nisd 守护进程模拟一个 NIS（YP）DNS 解析器服务。这并不是 rpc.nisd 或 mk_nisd的缺省设置。

注： 如果使用 -I 或 -B 标志，那么在系统重新引导后因为使用了 -a、-y 和 -b 标志的设置仍然是 rpc.nisd 的缺省行为。恢复设置的唯一方法是执行 rm_nisd，然后再次执行 mk_nisd。

示例

  1. 要修改 /etc/rc.nfs 文件以在下一次系统重新启动时调用 rpc.nisd 守护进程，请输入： 

    mk_nisd -I
  2. 要不带 DES 认证启动 rpc.nisd 守护进程和修改 /etc/rc.nfs 文件以在重新引导时不带 DES 认证调用 rpc.nisd 守护进程: 

    mk_nisd -B -s

文件

/etc/rc.nfs 包含用于 NFS 和 NIS 守护进程的启动脚本。
