mknfsmnt 命令

用途

从一台 NFS 服务器安装一个目录。

语法

/usr/sbin/mknfsmnt -f PathName -d RemoteDirectory -h RemoteHost [ -t { rw | ro } ] [ -m MountTypeName ] [ -w { fg | bg } ] [ -X | -x ] [ -S | -H ] [ -Y | -y ] [ -Z | -z ] [ -e | -E ] [ -a | -A ] [ -j | [ -J ] [ -q | [ -Q ] [ -g | [ -G ] [ -s | -n ] [ -I | -B | -N ] [ -r TimesToRetry ] [ -R NumRetrans ] [ -b ReadBufferSize ] [ -c WriteBufferSize ] [ -o TimeOut ] [ -P PortNumber ] [ -u AcRegMin ] [ -U AcRegMax ] [ -v AcDirMin ] [ -V AcDirMax ] [ -T AcTimeOut ] [ -p NumBiods ] [ -K { any | 2 | 3 } ] [ -k { any | tcp | udp } ]

描述

mknfsmnt 命令构造了 一个将附加到 /etc/filesystems 文件的条目，从而使文件系统可用于安装。 如果安装是永久的，入口将会保留。 如果安装是临时的， mount 命令将会直接使用标志。 如果是软安装，当服务器不响应时，系统将会返回一个错误。 如果是硬安装，客户机会一直试直到服务器响应为止。 缺省为硬安装。

标志

-A 这个文件系统的 /etc/filesystems 项指明当系统重启时，应该自动挂装这个文件系统。
-a 这个文件系统的 /etc/filesystems 项指明当系统重启时，不应该自动挂装这个文件系统。这是缺省情况。
-B 为 /etc/filesystems 文件添加一个入口， 准备安装这个文件系统。 这个标志是缺省的。
-b ReadBufferSize 使用 ReadBufferSize 变量表明读缓冲的大小以字节为单位。
-c WriteBufferSize 使用 WriteBufferSize 变量表明写缓冲的大小以字节为单位。
-d RemoteDirectory 说明目录安装到指定路径名。
-E 在硬安装时允许键盘中断。
-e 在硬安装时拒绝键盘中断。这是缺省的。
-f PathName 说明远程目录的安装点。
-G 使在文件系统上创建的文件或目录继承父目录的组 ID。
-g 使在文件系统上创建的文件或目录不继承父目录的组 ID。 这是缺省情况。
-H 创建一个硬安装，它使得客户机一直重试直到服务器响应。 这是缺省情况。
-h RemoteHost 指明导出目录的 NFS 服务器。
-I 为 /etc/filesystems 文件添加一个入口。 目录没有安装。
-J 指明在这次安装使用了 acls。
-j 指明这次安装不使用 acl。 这是缺省情况。
-K 指明 NFS 安装的版本。只有在 AIX 4.2.1 或以后的版本才使用这个标志。选项是：
any
使用 mount 命令来确定正确的匹配，之前尽量安装可用的最高版本的 NFS。
2
说明 NFS 2.版本
3
说明 NFS 3.版本
-k 指明安装的传输协议。只有在 AIX 4.2.1 或以后的版本才使用这个标志。选项为：
any
使用 mount 命令来选择使用的协议。最好使用 TCP 协议。
tcp
指明是 TCP 协议。
udp
指明是 UDP 协议。
-m MountTypeName 指明安装的文件系统类型。在 /etc/filesystems 文件中的 type 变量指明了文件系统类型。当使用 mount -t MountTypeName 命令，当前所有的类型为 MountTypeName 的未安装的文件系统都会被安装。
-N 使用指定的安装目录，但不改变 /etc/filesystems 文件。
-n 指示安装不使用一个更安全的协议。这个标志是缺省的。
-o TimeOut 用 TimeOut 变量指明 NFS 的超时时间，单位是十分之一秒。
-P PortNumber 指明服务器的因特网协议端口号。
-p NumBiods 指明允许在一个特定文件系统工作的 biod 守护进程的数目。 客户机需要的 biod 守护进程句柄和缺省的守护进程数目为6（六）。
-Q 不允许交换路径信息，并且信息在安装 NFS V2时可用。 在 NFS 服务器上 rpc.mountd 需要安装版本2。
-q 如果安装 NFS 版本2，不允许交换路径信息。 这是缺省的。
-r TimesToRetry 指明再安装的次数。缺省值为 1000。
-R NumRetrans 对软安装而言，指明当请求不被服务器认可时，请求传输的数目。 如果在 NumRetrans 次传输后，请求仍被拒绝，客户机放弃请求。 如果没有指定这个标志，缺省使用值为3。
-S 创建一个软安装，这意味着当服务器不响应时，系统返回一个错误。
-s 指示安装使用一个更安全的协议。
-T AcTimeOut 将标准的文件和目录的最小和最大允许数设为 Actimeo 变量指定的秒数。 如果指定了这个标志，另一个高速缓冲属性时间将被重设。
-t Type 指明目录或是读/写的或是只读的。
rw
可读写的安装目录。这种类型对系统来说是缺省的。
ro
只读的安装目录。
-U AcRegMax 在修改文件后，保持高速缓存属性不大于 AcRegMax 变量指定的秒数。
-u AcRegMin 在修改文件后，保持高速缓存属性至少为 AcRegMin 变量指定的秒数。
-V AcDirMax 在修改目录后，保持高速缓存属性不大于 AcDirMax 变量指定的秒数。
-v AcDirMin 在修改目录后，保持高速缓存属性至少为 AcDirMin 变量指定的秒数。
-w Location 指明应安装在哪里。Location 变量的值为下列之一：
fg
尝试在前台安装。这是缺省值。
bg
尝试在后台安装。如果指定了后台安装，并且安装目录失败了，安装将以后台方式重试。
-x 指明服务器不支持长设备号。

从 NFS 服务器安装时，使用这个标志，将不能正确处理32位长的设备号。
-X 指明服务器支持长设备号。这是缺省情况。
-y 指明在这个文件系统中，不允许运行 suid 和 sgid 程序。
-Y 指明在这个文件系统中，允许运行 suid 和 sgid 程序。 这是缺省情况。
-z 指明在安装过程中，不允许设备访问；也就是说，在安装点上不能打开设备。
-Z 指明在安装过程中，允许设备访问。这是缺省情况。

示例

添加一个远程目录安装，输入：

mknfsmnt -f /usr/share/man -d /usr/share/man -h host1

在这个例子中，mknfsmnt 命令将远程目录 /usr/share/man 安装到驻留在 host1 的 /usr/share/man 目录上。

文件

/etc/filesystems 在系统重启过程中，列出要安装的远程文件系统。
