mkclient 命令

用途

取消 /etc/rc.nfs 文件中对 ypbind 守护进程的项的注释，并启动 ypbind 守护进程以配置客户机。

语法

/usr/sbin/mkclient [ -I | -B | -N ] [ -S server]

描述

mkclient 命令取消了 /etc/rc.nfs 文件项的注释，以启动 ypbind 守护进程来配置客户机。mkclient 命令使用适当的系统资源控制器（SRC）命令来启动 ypbind 守护进程。

您可以使用基于 Web 的系统管理器（wsm）中的文件系统应用程序来更改文件系统特性。

您也可以使用“系统管理接口工具”（SMIT）smit mkclient 快速路径来运行此命令。

标志

-I 取消 /etc/rc.nfs 文件中启动守护进程 ypbind 的项的注释。此项使得守护进程 ypbind 在下一次系统重新启动时运行。
-B 取消 /etc/rc.nfs 文件项的注释，并且启动 ypbind 守护进程。此标志是缺省值。
-N 使 startsrc 命令启动 ypbind 守护进程。此标志不会影响 /etc/rc.nfs 文件。
-S 指定使用 NIS server，而不是单个广播。只有在直接连接到客户机的网络上没有 NIS 服务器时，此标志才可用。

示例

修改 /etc/rc.nfs 文件，以便在下一次系统重新启动时启动 ypbind 守护进程，请输入：

mkclient -I

文件

/var/yp/domainname 目录 包含 NIS 域的 NIS 映射。
/etc/rc.nfs 包含 NFS 和 NIS 守护进程的启动脚本。
