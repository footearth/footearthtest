inulag 命令

用途

作为子例程的前端来管理许可证协议。

语法

inulag -r [ -n FilesetName | -s FileName | -p Product ] [ -d Description [ -m MessageSpecification ]] -f File

inulag -l | -q [ -c | -v ] [ -n FilesetName | -s FileName | -p Product | -a ]

inulag -u [ -n FilesetName | -s FileName | -p Product ]

inulag -A

inulag -D

描述

inulag 命令管理软件许可证协议。基本格式是许可证协议注册、许可证协议列表、许可证协议失效、许可证协议验证和许可证协议的重新验证。

-r 标志管理以 installp 命令安装的文件集或者是通过另外一个安装程序安装的独立安装产品的许可证协议注册。当许可证协议注册后，到总是以独立安装产品安装的文件的路径必须用 -s 标志指定。

-l 标志列出软件许可证协议注册。如果指定了 -c 标志，那么将显示到软件许可证协议的路径而不是文件的内容。

-q 标志查询软件许可证协议是否存在。 如果许可证协议存在，将返回为 0 的返回码。如果 -a 标志也被指定的话，那么将返回为 0 的返回码（如果有一个暂挂许可证协议）。

-u 标志为文件集或者独立安装产品出去软件许可证协议的列表。

-D 标志在下次重新引导机器的基础上强制软件许可证协议重新验证。

标志

-a 和 -l 标志一起用来显示有暂挂许可证协议的产品。
-A 为所有暂挂许可证协议注册协议。
-c 与 -l 标志一起用来显示以冒号隔开的列表。不能和 -v 标志一起用。
-d Description 指定许可证应用的产品或文件集的缺省描述。
-D 强制在下次重新引导后重新验证所有许可证协议。
-f File 为许可证协议指定路径名规格。在这个规格中的‘%L’是当前语言环境的一种代替模式。en_US 是缺省语言环境。在规格中的“%l”和语言环境的前两个字符匹配，除非当前语言环境是 zh_CN，在这种情况下语言环境指定的所有五个字母都要用到。
-l 列出软件许可证协议。
-m MessageSpecification 为“编目，集合数，消息数”格式的转换描述指定消息编目。
-n FilesetName 指定在由许可证协议管理的软件重要产品数据库中注册的文件集名称。
-p Product 指定产品标识，它是唯一标识某个产品的但不能翻译的字母数字字符串。
-q 查询许可证协议。不要显示输出。如果许可证协议存在就返回 0 的值。-q 标志可以和其它标志一起使用来查询特定的许可证协议或者暂挂许可证协议。
-r 注册一个软件许可证协议。需要用 -f 标志来显示协议文件的路径，用 -n 标志或者用 -s 标志来指出文件集名或者包含符合协议的软件的签名文件。-r 标志不能和 -l 或者 -q 或者 -u 标志一起使用。 许可证协议在系统安装和 NIM SPOT 安装时登记为暂挂（状态='P'），除非环境变量 ACCEPT_LICENSES 设置为 yes。
-s FileName 为安装的软件指定唯一的签名文件，这个软件标识了没有在由软件协议许可证管理的软件重要产品数据库中注册的软件。它是给没有注册进软件重要产品数据库的软件产品使用的。这个格式是用来标识已安装但是没有在软件重要产品数据库中注册的软件。FileName 包括到文件的全路径。
-u 删除一个许可证协议。这实际上并不会删除许可证协议文件，相反它将与文件集相关的许可证协议状态更改为不活动。不活动的许可证协议不需要重新同意，但是当列出已经安装的软件许可证时，它们不会显示出来。
-v 用 -l 标志来查看详细的列表。不能和 -c 标志同时使用。

安全性

协议数据库仅对 root 用户可写。因此，除了 -l 标志外，所有的标志只能被有 root 用户权限的用户使用。
