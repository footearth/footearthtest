lppmgr 命令

用途

管理一个现有的 installp 映像源。

语法

lppmgr -d DirectoryOrDevice [ -r | -m MoveDirectory ] { [ -x ] [ -X ] [ -l ] [ -u ] [ -b ] [ -k LANG ] } [ -p ] [ -t ] [ -s ] [ -V ] [ -D ]

描述

lppmgr 设计用于在现有的 installp 映像源（在 NIM 环境中也称为 lpp_source ）上执行以下功能：

  1. 删除重复的更新（-u 标志）。
  2. 删除重复的基级别（-b 标志）。
  3. 删除那些与相同文件集的基相同等级的更新。这样的更新会产生冲突从而导致安装失败（-u 标志）。
  4. 删除消息以及指定语言之外的语言环境文件集（-k 标志）。
  5. 删除被取代的文件集（-x 标志）。
  6. 从一个 NIM lpp_source 资源删除非系统映像（-X 标志）。

缺省情况下，lppmgr 将列出由上述例程过滤的所有映像。“-r”标志能用于删除过滤的映像，而“-m”标志能用于移动映像到另一个位置。

注:
lppmgr 并非意在替换 bffcreate、安装任何东西或者处理已经安装的文件集。 它也不是试图处理除上述那些之外的任何问题。使用“-X”标志之前，必须好好理解 NIM、系统映像（在 NIM 中称为 SIMAGES）和 NIM lpp_source 资源的工作方式。

标志

-b 使 lppmgr 为基级别复制过滤。
-D 指定调试方式。该标志用于调试 lppmgr 脚本。
注:
调试。它产生大量的输出并且大大降低 lppmgr 性能。对于正常操作它没有用。
-d DeviceOrDirectory 指定 installp 映像驻留的设备或目录。现在它可以是任意目录、NFS 安装点或者 cdrom 设备。如果此目录不可写，应该使用 -t 标志。如果操作的目标是一个 NIM lpp_source 资源，必须指定 lpp_source 的位置（请参阅 lsnim 命令）。该标志对所有的操作都需要。
-k LANG 对 LANG 指定的语言只保存消息和语言环境映像。其它所有语言被过滤。
-l 仅列出被过滤的映像。缺省情况下，lppmgr 将仅仅列出所有被过滤的映像文件，除非指定了“-r”或者“-m”标志。“-l”标志将覆盖“-r”或“-m”标志。
-m Directory 将过滤文件移动到 Directory。由 Directory 指定的位置可以是任意可写的目录路径。该标志不能与“-r”标志一起使用。
-p 指定提示方式。当移动或删除文件的时候提示。
-r 删除已经被 lppmgr 过滤的文件。
注:
如果没有指定提示标志（-p），lppmgr 将在不再与用户进一步交互的情况下删除所有的过滤文件。该标志不能与“-m”标志一起使用。
-s 打印空格使用信息。它将打印一个特定文件集正在使用的空格数目，以及想要知道的空格总数。为文件元数据添加一些缓冲区空间。
-t 指定 lppmgr 将不会重建 .toc 文件。它可以用来进行快速浏览而不用耗时重建整个 .toc 文件。而且，此为只读设备所需要的。
-u 使 lppmgr 过滤重复的更新和更新与相同文件集的基相同等级而产生的冲突。
-V 指定详细方式。lppmgr 将在某些情况下给出更多的输出。
-x 使 lppmgr 为被取代的更新过滤。
-X 从 NIM lpp_source 资源过滤非系统映像。

退出状态

0
所有 lppmgr 相关操作成功完成。
>0
出现错误。

安全性

只有 root 用户能够执行 lppmgr。

示例

  1. 要在映像源目录 /myimages 列出所有重复和冲突的更新，请输入： 
    lppmgr -d /myimages -u
  2. 要在映像源目录 /myimages 删除所有重复和冲突的更新，请输入： 
    lppmgr -d /myimages -u -r
  3. 要在提示方式下删除所有重复和冲突的更新、重复的基级别以及除了“en_US”之外的所有消息／语言环境文件集，请输入： 
    lppmgr -d /myimages -purb -k en_US
  4. 要从 NIM lpp_source 的位置 /lpps/433 移动所有被取代的更新映像和非 SIMAGES 到目录 /backups，请输入： 
    lppmgr -d /lpps/433 -x -X -m  /backups

文件

/usr/lib/instl/lppmgr 包含 lppmgr 命令。
