mgmtsvr 命令

用途

mgmtsvr - 显示或设置“受管”节点的管理服务器项。

语法

mgmtsvr [-h] [-k] [-v | -V] [-N] [-n hostname] [host | -d]

描述

mgmtsvr 命令在“受管”节点上运行以显示或设置节点的管理服务器。

标志

-d
删除管理服务器项。
-h
将命令的用法语句写到标准输出中。
host
指定将要管理这个节点的 CSM 管理服务器的主机名或 IP 地址。如果没有指定主机，并且没有指定 -d 标志，则会显示当前设置。在这种情况下，能够通过检查退出状态来区分节点和管理服务器。
-k
在节点和管理服务器之间交换 RSCT 公用密钥、主机名和节点标识符。如果任何机器的主机名、节点标识符或公用密钥已更改，则仅使用该标志。当使用该标志时，必须确信您的网络以确保恶意的机器不会插入它的公用密钥，来代替实节点的公用密钥。
-n hostname
为该节点指定 CSM 管理服务器已知的的主机名。将节点名传递给 updatenode 命令通常只需要用于诊断用途。updatenode 命令使用 -n 标志来确保当节点上的 hostname 值解析成与管理服务器已知的名称不同的名称时进行正确的操作。缺省值是 /bin/hostname。在使用 updatenode 命令并带有 -n 标志之前，必须运行 chnode nodename AllowManageRequest=1。
-N
显示本地节点名和管理服务器的值。只有在管理服务器的主机名或 IP 地址没有指定的情况下，该标志才有效。
-v | -V
将命令的详细消息写到标准输出中。

参数

无。

安全性

该命令要求 root 用户至群集管理服务器的访问权。

退出状态

0
这仅是“受管”节点，并且已设置管理服务器值。
1
标志组合无效。
11
这仅是“受管”节点，并且未设置管理服务器值。
21
这个节点是其本身的管理服务器。
22
这个节点也是一个单独群集的管理服务器。
23
这台机器是 CSM 管理服务器和节点，但该节点本身当前并没有设置管理服务器。
31
这台机器仅仅是 CSM 管理服务器。
101
这台机器既不是 CSM 管理服务器，也不是节点。

环境变量

DSH_FANOUT
设置并发远程 shell 命令的最大数。也可由 -f 标志设置。
DSH_LIST
指定一个文件，该文件包含构成节点列表的节点集的定义。
DSH_LOG
设置为日志文件的绝对路径和文件名后启用日志记录。此变量将由 --log 和 --noLogging 标志覆盖。
DSH_PATH
设置远程节点上所用的路径。如果未设置 DSH_PATH，将使用远程 shell 的缺省路径。例如，DSH_PATH=$PATH 将远程节点上的路径设置为源节点上所用的相同路径。
DSH_REMOTE_CMD
指定要使用的可执行远程 shell 的路径，而不是缺省路径。
DSH_REMOTE_OPTS
包含将此命令转发到远程节点时在远程命令中指定的选项。
DSH_REPORTS_DIRECTORY
设置为报告目录路径后启用报告和输出文件。此变量将由 --reports 和 --noReporting 标志覆盖。
DSH_TIMEOUT
指定等待来自远程 shell 的响应的时间（以秒为单位）。也可以使用 -t 标志设置。
WCOLL
等同于 DSH_LIST。

标准错误

使用 dshbak 过滤器并生成了标准错误消息时，有关标准错误的所有错误消息都将在所有标准输出消息之前显示。无论是否使用 -c 标志，这都成立。

示例

要查询这个节点的当前 CSM 管理服务器，请输入： 

mgmtsvr

位置

/opt/csm/bin/mgmtsvr
