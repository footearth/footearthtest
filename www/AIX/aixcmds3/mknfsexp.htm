mknfsexp 命令

用途

导出一个到 NFS 客户机的目录。

语法

/usr/sbin/mknfsexp -d Directory [ -f Exports_File ] [ -t [ { rw | ro | rm -h HostName [ , HostName ... ] } ] ] [ -a UID ] [ -r HostName [ , HostName ... ] ] [ -c HostName [ , HostName ... ] ] [ -s | -n ] [ -I | -B | -N ] [ -P | -p ]

描述

mknfsexp 命令使用了指定的标志和参数 并构造了一行（其句法对于 /etc/exports 文件是正确的）。如果命令 由 -B 标志调用，会在 /etc/exports 文件中添加一个项并且 exportfs 命令被调用以导出指定的目录。同时， -I 标志增加了一项到 exports 文件中并且不导出目录，或者 -N 标志不在 exports 文件中增加一项而是导出目录。

标志

-d Directory 指定要导出或是改变的目录。
-f Exports_File 如果不是 /etc/exports 文件，则要指定使用的导出文件的全路径名称。
-t Type 指定目录是读／写、只读或是完全读。 type 变量可能的数值为：
rw
导出读／写目录。这是系统的缺省值。
ro
导出只读目录。
rm
导出完全读目录。如果选择后，-h 标志必须用来指定已经得到读／写许可权的主机。
-h HostName [ , HostName ] ... 指定哪台主机有访问目录的读／写权。这个选项仅当导出文件是完全读类型的时候才是正确的。
-a UID 当请求来自于未知的用户使用 UID 变量作为有效的用户标识符。此项缺省的值为 －2。
注:
NFS 服务器一般将 root 用户 （UID 0）视为未知的，除非它们包含在 root 选项中。设置 UID 的值为 -1 ，这样可以禁用匿名访问。
-r HostName [ , HostName ] ... 分配给在指定主机上的 root 用户访问目录的权力。 缺省是没有主机授权以 root 访问权。
-c HostName [ , HostName ] ... 给每个所列的客户机分配安装访问权。客户机可以是主机或是网络组。 缺省的是允许所有主机的访问。
-s 当访问目录的时候，需要客户机使用更安全的协议。
-n 不需要客户机使用更安全的协议。这个标志是缺省的。
-B 在 /etc/exports 文件中增加一个项，并且 exportfs 命令执行导出目录 这个标志是缺省的。
-I 在 /etc/exports 文件中增加一个项，这样下一次在系统重启时运行 exportfs 命令，目录会被导出。
-N 不在 /etc/exports 文件中增加一个项，使用正确的参数的 exportfs 命令会导出目录。
-P 指定导出的目录会是个公共的目录。 这个标志仅应用于 AIX 4.2.1 或其之后的。
-p 指定导出的目录不是公共的目录。 值个标志仅应用于 AIX 4.2.1 或其以后的。

示例

  1. 为了导出一个具有只读许可权的目录，输入： 

    mknfsexp -d /usr -t ro

    在此例中， mknfsexp 命令在只读许可权下导出 /usr 目录。

  2. 使用完全读许可权和指定主机的安全协议导出目录，输入： 

    mknfsexp -d /home/guest -t rm -h bighost,littlehost -s

    此例中， mknfsexp 命令用完全读许可权导出 /home/guest 目录，使用更安全的协议。

  3. 用读／写许可权导出目录到指定的网络组和指定的主机，并在下一次系统重新启动时使导出可用，输入： 

    mknfsexp -d /usr -t rw -c host1,host3,grp3 -I

    在上面的例子中，mknfsexp 命令 导出 /usr 目录并分配读和写许 可权给 host1、 host2 和 grp3。 -I 标志使得在下一次系统重新启动时改变被实现。

  4. 用只读许可权导出目录到除 /etc/exports 外的导出文件，输入： 

    mknfsexp -d /usr -t ro -f /etc/exports.other

    在上面的例子中，mknfsexp 命令用只读许可权导出 /usr 目录到 /etc/exports.other 文件中。

文件

/etc/exports 列出服务器可以导出的目录。
