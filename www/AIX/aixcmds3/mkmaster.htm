mkmaster 命令

用途

执行 ypinit 命令并启动 NIS 守护进程以配置主控服务器。

语法

/usr/sbin/mkmaster [ -s HostName [ ,HostName ... ] ] [ -O | -o ] [ -E | -e ] [ -P | -p ] [ -U | -u ] [ -C | -c ] [ -I | -B | -N ]

描述

如果当前没有设置系统的域名，则 mkmaster 命令会调用 ypinit 命令为当前的域创建 NIS 映射。在 ypinit 命令成功完成之后，对于 ypserv 命令、yppasswdd 命令、ypupdated 命令和 ypbind 命令，mkmaster 命令取消了 /etc/rc.nfs 文件中项的注释。

您可以使用基于 Web 的系统管理器（wsm）中的文件系统应用程序来更改文件系统特性。您也可以使用“系统管理接口工具”（SMIT）smit mkmaster 快速路径来运行此命令。

标志

-s HostName [,HostName ... ] 为主控服务器指定从主机名。这些从主机名必须在主控服务器配置之后再配置。mkmaster 命令自动将当前主机添加到此列表中。
-O 覆盖此域的现有映射。
-o 防止覆盖此域的现有文件。这个标志是缺省值。
-E 在创建新的映射时，如果遇到错误，则防止进一步的操作。对于 ypinit 命令和 mkmaster 命令，这是正确的。这个标志是缺省值。
-e 如果遇到错误，不要从 ypinit 命令和 mkmaster 命令中退出。
-P 和 ypserv 守护进程一起启动 yppasswdd 守护进程。
-p 禁止启动 yppasswdd 守护进程。这个标志是缺省值。
-U 和 ypserv 守护进程一起启动 ypupdated 守护进程。
-u 禁止启动 ypupdated 守护进程。这个标志是缺省值。
-C 和 ypserv 守护进程一起启动 ypbind 守护进程。这个标志是缺省值。
-c 禁止启动 ypbind 守护进程。
-I 使用 mkmaster 命令更改 /etc/rc.nfs 文件以在下一次系统重新启动时启动相应的守护进程。当激活此命令时，执行 ypinit 命令。
-B 执行 ypinit 命令，取消文件 /etc/rc.nfs 中的项的注释，并启动守护进程。这个标志是系统缺省值。
-N 执行 ypinit 命令并启动相应的守护进程，不会更改 /etc/rc.nfs 文件。

示例

要执行 ypinit 命令，覆盖当前域中的任何现有的映射，并制作 host1 和 host3 从服务器，请输入：

mkmaster -s host1,host3 -O -p -u -B

此命令不会启动 yppasswdd 守护进程或 ypupdated 守护进程。

文件

/var/yp/domainname directory 包含 NIS 域的 NIS 映射。
/etc/rc.nfs 包含 NFS 和 NIS 守护进程的启动脚本。
