mkserver 命令

用途

把子服务器定义添加到 Subserver 对象类。

语法

mkserver -c CodePoint -s Subsystem -t Type

描述

mkserver 命令把子服务器定义添加到 Subserver 对象类

标志

-c CodePoint 指定标识子服务器的 CodePoint 整数。子系统通过这个值来识别子服务器。如果该 CodePoint 值对于此子系统来说已经存在，则 mkserver命令是失败的。CodePoint 存储的限制与短整数相同（1 到 32,768）。
-s Subsystem 给子服务器所属的子系统指定唯一标识它的名称。如果在子系统对象类中 Subsystem 名不是已知的，或者如果 Subsystem 名是子系统对象类中已知的子系统，但是它使用信号作为通信方法，则 mkserver 命令是失败的。
-t Type 指定唯一标识子服务器的名称。如果 Type 名在 Subserver Type 对象类中是已知的，则 mkserver 命令是失败的。

安全性

审计事件：如果审计子系统已经进行了正确地配置并已启用，则 mkserver 命令将在每次执行时生成以下审计记录（事件）：

事件 信息
SRC_Addserver 已添加的审计日志子系统列表和整个“对象数据管理”记录。

有关如何恰当的选择和分组审计事件以及如何配置审计事件数据收集的更多详细信息，请参阅《AIX 5L V5.2 安全指南》中“设置审计”。

示例

要添加子服务器定义，输入：

mkserver  -s srctest  -t tester  -c 1234

这把子服务器定义添加到 Subserver Type 对象类中，且所有者子系统为 srctest，子服务器代码点为 1234。

文件

/etc/objrepos/SRCsubsys 指定 SRC Subsystem Configuration 对象类
/etc/objrepos/SRCsubsvr 指定 SRC Subserver Configuration 对象类
