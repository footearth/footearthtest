trcdead 命令

用途

从系统转储映像中抽取跟踪缓冲区。

语法

trcdead [  -oFileName ] DumpImage

描述

trcdead 命令从系统转储中抽取八个活动的跟踪通道。如果系统停机而跟踪工具仍然是活动的，则内部跟踪缓冲区的内容被捕获。如果使用了 -o 标志，trcdead 命令将将跟踪事件数据从转储中抽取出来并写进指定的文件中。如果没有选择该标志，该命令将写进跟踪日志文件。

如果使用 trace -C 为多个 CPU 请求多 CPU 跟踪缓冲，则这些缓冲区不会被转储。

使用 trcrpt 命令来格式化一个来自跟踪日志文件的报告。

标志

-oFileName 指定要写入数据的文件。

示例

注：要决定哪一个示例更适合你的计算机系统，请使用 sysdumpdev 命令来显示当前所指定的转储设备。
  1. 要从一个转储映像中抽取信息，写入一个名为 trace_extract 的文件，请输入： 

    trcdead -o trace_extract /var/adm/ras/dumpfile
  2. 要从一个转储映像中抽取信息，写入一个设备，请输入： 

    trcdead /dev/hd7

文件

/usr/bin/trcdead 包含 trcdead 命令。
/var/adm/ras/dumpfile 包含缺省系统转储文件。
/var/adm/ras/trcfile 包含缺省系统跟踪日志。
