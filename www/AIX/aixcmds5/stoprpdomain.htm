stoprpdomain 命令

用途

将一个联机的对等域脱机。

语法

stoprpdomain [-f] [-h] [-TV] peer_domain

描述

stoprpdomain 命令将所有目前联在对等域上的节点脱机。不从节点上除去对等域定义。

该命令必须在联入对等域的节点上运行。如果该命令在与对等域脱机的节点上运行，将不执行任何操作。

必须使用 -f 标志以覆盖子系统对对等域脱机请求的拒绝。 如果对等域资源正忙（例如共享磁盘），则子系统可能会拒绝这种请求。 在这种情况下指定 -f 标志向子系统表明对等域必须脱机，而不管资源状态如何。

标志

-f
强制子系统接受停止请求，否则子系统将不会接受。
-h
将命令的用法语句写到标准输出中。
-T
将命令的跟踪消息写到标准错误中。仅供您的软件服务组织使用。
-V
将命令的详细消息写到标准输出中。

参数

peer_domain
指定要脱机的联机对等域的名称。

安全性

stoprpdomain 命令的用户需要将 IBM.PeerDomain 资源类的写权限写到每个定义在对等域的节点中。 缺省情况下，在对等域中任何节点的 root 用户通过配置资源管理器拥有对此资源类的读写权限。

退出状态

0
命令成功运行。
1
RMC 出错。
2
命令行界面脚本出错。
3
在命令行上输入了不正确的标志。
4
在命令行上输入了不正确的参数。
5
不正确的命令行输入导致出错。
6
对等域定义不存在。

环境变量

CT_CONTACT
确定与资源监视和控制（RMC）守护进程发生会话的系统。 当 CT_CONTACT 设置为主机名或 IP 地址时，该命令会与指定主机上的 RMC 守护进程联系。 如果未设置 CT_CONTACT，该命令会与运行它的本地系统的 RMC 守护进程联系。 RMC 守护进程会话的目标和管理作用域确定了处理的资源类或资源。

限制

该命令必须在联入对等域的节点上运行。

标准输入

当指定 -f "-" 标志时，该命令从标准输入中读取一个或多个节点名。

标准输出

当指定 -h 标志时，将该命令的用法语句写到标准输出中。 所有详细消息都写到标准输出中。

标准错误

所有跟踪消息都写到标准错误中。

示例

在这些示例中，nodeA 是定义的节点之一，并且联在 ApplDomain 上。

  1. 要使 ApplDomain 脱机，请在 nodeA 上运行此命令： 
    stoprpdomain ApplDomain
  2. 要使 ApplDomain 脱机，同时确保任何子系统不会拒绝停止请求，请在 nodeA 上运行此命令： 
    stoprpdomain -f ApplDomain

位置

/usr/sbin/rsct/bin/stoprpdomain
包含 stoprpdomain 命令

文件

/etc/services 文件已修改。
