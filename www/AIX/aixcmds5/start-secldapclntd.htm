start-secldapclntd 命令

用途

start-secldapclntd 脚本用于启动 secldapclntd LDAP 客户机守护进程。

语法

/usr/sbin/start-secldapclntd [ -C CacheSize ] [ -p NumOfThread ] [ -t CacheTimeOut ] [ -T HeartBeatIntv ]

描述

如果 secldapclntd 守护进程未运行，则 start-secldapclntd 脚本启动此程序。如果 secldapclntd 守护进程已经运行，则该脚本不进行任何操作。该脚本也在启动 secldapclntd 守护进程之前，从以前的 secldapclntd 守护进程中清理端口映射程序的注册（如果有的话）。这就防止了由于端口映射程序注册失败而引起的新建守护进程的启动故障。

标志

在缺省情况下，secldapclntd 守护进程在启动时读取 /etc/security/ldap/ldap.cfg 文件中指定的配置信息。如果当启动 secldapclntd 进程时，下列选项出现在命令行中，则这些来此命令行的选项将覆盖 /etc/security/ldap/ldap.cfg 文件中的值。

-C CacheSize 将 secldapclntd 守护进程使用的最大高速缓存项设置为 CacheSize 项数。对于用户的高速缓存，有效范围是 100-10,000。缺省值是 1000。组高速缓存项是用户高速缓存项的 10%。
-p NumOfThread 将 secldapclntd 守护进程使用的线程数设置为 NumOfThread 个线程。有效范围是 1-1000。缺省值是 10。
-t CacheTimeout 将高速缓存设置为在 CacheTimeout 秒后失效。有效范围是 60-3600 秒。缺省值是 300 秒。
-T HeartBeatIntv 设置客户机与 LDAP 服务器之间波动信号的时间间隔。有效值是 60-3,600 秒。缺省值是 300。

示例

  1. 要启动 secldapclntd 守护进程，请输入： 
    /usr/sbin/start-secldapclntd
  2. 要启动使用 20 个线程和高速缓存超时值为 600 秒的 secldapclntd，请输入： 
    /usr/sbin/start-secldapclntd -p 20 -t 600
    建议您在 /etc/security/ldap/ldap.cfg 文件中指定这些值，这样您就可以在每次启动 secldapclntd 进程时使用这些值。
