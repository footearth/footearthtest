smimpservercert 命令

用途

导入服务器证书。

语法

smimpservercert dir { -a | -s server_name}

描述

smimpservercert 命令把一个服务器证书（*.cert 文件）导入到服务器专用密钥文件（*.privk）中，生成一个专用密钥环文件（*.privkr）。您可以导入一个服务器的证书，或所有和指定目录 dir 中的专用密钥文件相匹配的证书。

使用 /usr/websm/bin/wsm 命令访问图形界面。快速路径是 wsm 系统。

标志

dir 证书申请（*.certreq 文件）和专用密钥（*.privk 文件）所在的目录，以及写入专用密钥环（*.privkr 文件）的目录。
-a 处理所有和指定目录 dir 中专用密钥文件（*.privk）相匹配的证书（*.certreq 文件）。每一个证书 S.cert 都要导入到专用密钥文件 S.privk 中，生成指定 dir 目录中的专用密钥环文件 S.privkr。
-s server_name 服务器的 TCP/IP 的全名，它的证书 server_name.cert 要导入到其专用密钥文件 server_name.privk 中，生成指定 dir 目录中的专用密钥环文件 server_name.privkr。

示例

smimpservercert /usr/websm/security/tmp S101.IBM.COM

smimpservercert /usr/websm/security/tmp -a

相关信息

smgenkeycr、smgenprivkr、sminstkey、smlistcerts 以及 smserverprop 命令。
