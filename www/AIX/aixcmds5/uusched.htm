uusched 守护进程

用途

“基本联网实用程序”（BNU）文件传输程序的调度工作。

语法

uusched [  -uDebugLevel ] [  -xDebugLevel ]

描述

uusched 守护进程调度“基本联网实用程序”（BNU）文件传输程序的工作。它调度在 /var/spool/uucp/SystemName 目录中排队的文件传送。调度守护进程首先随机化该工作，然后启动传送文件的 uucico 守护进程。

uusched 守护进程通常由 uudemon.hour 命令启动。此命令是一个 shell 过程，定期根据 /var/spool/cron/crontabs/uucp 文件中的指令由 cron 守护进程来运行。

为了进行调试，uusched 守护进程也可以从命令行启动。

注：当启动 uusched 守护进程时，您必须处于 /usr/sbin/uucp 目录中，或者您必须用全路径名 /usr/sbin/uucp/uusched 来启动该守护进程。

标志

-uDebugLevel 此标志将作为 -xDebugLevel 标志传递给 uucico 守护进程。DebugLevel 变量是从 0 到 9 的数字，缺省值是 5。越大的数字将给出越详细的调试信息，该信息显示在本地系统的屏幕上。
-xDebugLevel 从 uusched 守护进程输出调试消息。DebugLevel 变量是从 0 到 9 的数字，缺省值是 5。越大的数字将给出越详细的调试信息，该信息显示在本地系统的屏幕上。

示例

要从命令行启动 uusched 守护进程，请输入：

/usr/sbin/uucp/uusched &

这将启动 uusched 守护进程作为后台进程。（注意命令中包含了路径名。）

文件

/etc/locks /* 包含了防止对设备的多重使用和对系统的多重调用的锁定文件。
/usr/sbin/uucp/* 包含 uusched 守护进程和 BNU 配置文件。
/etc/uucp/Devices 包含有关可用设备的信息。
/etc/uucp/Maxuuscheds 限制已调度的作业。
/etc/uucp/Systems 描述可访问的远程系统。
/var/spool/cron/crontabs/uucp 调度 cron 守护进程，包括 uudemon.hour shell 过程的 BNU 作业。
/var/spool/uucp/SystemName /* 包含等待传送的文件。
