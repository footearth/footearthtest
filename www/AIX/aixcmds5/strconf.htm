strconf 命令

用途

查询流配置。

语法

strconf [  -t | -m module ]

描述

strconf 命令用来查询流的配置。当不使用任何标志时，它打印该流中的所有模块的列表以及顶端驱动程序。 该列表每行打印一个名称，其中打印的第一个名称是流上的最顶端模块，打印的最后一项是该驱动程序的名称。

注：strconf 命令只从标准输入中读取。

标志

-m Module 确定指定的模块是否在流上。如果该模块在流上，则 strconf 命令打印 yes 消息并返回 0 值。如果该模块不在流上，则 strconf 命令打印 no 消息并返回非零值。

-t 和 -m 标志是互斥的。
-t 只打印该流的最顶端模块（如果存在）。

参数

Module 指定要查看的模块。

示例

  1. 对于只有 ldterm 模块在 ports 驱动程序上推动的流，strconf 命令（无任何标志）将产生以下输出： 

    ldterm ports
  2. 输入以下命令来询问 ldterm 模块是否在该流上： 

    strconf -m ldterm

    该命令返回退出状态 0，同时产生以下输出： 

    yes
