updatevsdnode 命令

用途

updatevsdnode － 修改虚拟共享磁盘子系统的选项。

语法

updatevsdnode
-n {ALL | node_number [,node_number ...]}
 
{[-a VSD_adapter]
 
[-b min_buddy_buffer_size]
 
[-x max_buddy_buffer_size
 
[-s max_buddy_buffers]
 
[-M vsd_max_ip_packet_size]}
 
[-f] [-c cluster_name | NONE]

描述

使用 updatevsdnode 来修改虚拟共享磁盘子系统的选项。

注:
该命令只修改子系统的选项。要有效地配置虚拟共享磁盘，您首先必须取消配置所有的虚拟共享磁盘，卸装设备驱动程序，然后重新配置这些共享磁盘。

可以使用“系统管理接口工具”（SMIT）来运行该命令。要使用 SMIT，输入

smit vsd_mgmt

并选择设置／显示虚拟共享磁盘设备驱动程序操作参数选项或更新虚拟共享磁盘设备驱动程序节点参数选项。

标志

-n
指定节点的数目（您要该命令去更新这些节点的信息），或 RSCT 对等域中的所有节点。可以发出命令 /usr/bin/lsclcfg 以查找您正在运行的节点的节点号。
-a
指定用于虚拟共享磁盘与这个或这些节点的通信的适配器名称。您必须指定 ml0 作为适配器名称。
-b
指定服务器用来满足对虚拟共享磁盘的远程请求的最小伙伴缓冲区。该值必须是的 2 乘方并且大于或等于 4096。IBM 建议使用值 4096（4KB）。
-x
服务器将用来满足远程请求的最大伙伴缓冲区。该值必须是的 2 乘方并且大于或等于 min_buddy_buffer_size。推荐值为 262144（256KB）。该值在 RSCT 对等域中的所有节点上必须都是一样的。
-s
这是要分配的 max_buddy_buffer_size 缓冲区的数目。当首次装入时，虚拟共享磁盘的设备驱动程序将具有缓冲区初始的大小，然后将根据需要动态地分配和收回附加的空间。对于 32 位内核，推荐的起始值为 128 个 256KB 的缓冲区。推荐值为 2000 个 256KB 的缓冲区。

伙伴缓冲区仅在服务器上使用。在客户机节点上，您可能要把 max_buddy_buffers 设置为 1。

注:
statvsd 命令将指示远程请求是否在排队等待伙伴缓冲区。
-M
指定虚拟共享磁盘的最大 IP 消息大小，以字节为单位。该参数选项的推荐值为 61440（61KB）。
-f
指定该命令将通过重新配置 RSCT 对等域中所有节点上的一个或多个虚拟共享磁盘来强制更新虚拟共享磁盘子系统的选项，当前就是在这些节点上配置各个虚拟共享磁盘。
-c cluster_name | NONE
更改节点所属的群集。NONE 将把节点从群集中除去。
注:
cluster_name 只有对于 SSA（串行存储器体系结构）磁盘才是必需的。

参数

无

安全

您必须具有 root 用户权限以运行该命令。

退出状态

0
说明命令已成功完成。
非零
表明有错误发生。

限制

您必须从对等域中联机的节点发出该命令。要使对等域联机，请使用 startrpdomain 命令。要使现有对等域中一个特定的节点联机，请使用 startrpnode 命令。有关创建和管理 RSCT 对等域的更多信息，请参阅 RSCT: Administration Guide。

标准输出

当前的 RVSD 子系统运行级别。

示例

要在节点 3 上把伙伴缓冲区大小增加到 48 个最大的伙伴缓冲区，请输入：

updatevsdnode -n 3 -s 48

注:
必须从内核取消配置设备驱动程序并重新装入以使该更改生效。

位置

/opt/lpp/vsd/bin/updatevsdnode
