shconf 命令

用途

管理系统挂起检测参数。

语法

shconf -d

shconf -R -l Name

shconf {-D [-O ] | -E [-O ]} [-H] -l Name

shconf -l Name [-a Attribute=Value] ...

描述

shconf 命令用于显示或者指定优先级问题检测和丢失的 I/O 检测的参数。

对于优先级问题，用户可以指定下列描述的五个操作，并且对于每个操作，用户可以指定检测的优先级别、在较低或同级优先级处没有执行进程或线程时的超时、警告操作的终端设备以及 getty 操作：

pp_cmd 启动路径参数指定的命令。
pp_errlog 记录错误日志中的错误。
pp_login 启动由终端设备参数（term）指定的串行线路上最高优先级的 getty。
pp_reboot 重新引导系统。
pp_warning 显示由终端设备参数（term）指定的控制台上的警告消息。

对于丢失的 I/O，用户可以指定下面列出的操作以及 errlog（它在启动丢失的 I/O 检测时是自动的）。存在适用于所有启用操作的唯一超时。

lio_warning 显示终端设备参数（term）指定的控制台上的警告消息。
lio_reboot 创建系统转储并重新引导系统

标志

-d 不管是否启用优先级问题检测和丢失的 I/O 检测，都要显示。
-R 恢复检测的指定名称的缺省值。
-a Attribute=Value 指定用于更改特定的属性值的属性值对。
-D 显示检测的指定名称的缺省值。
-E 显示检测的指定名称的有效值。
-H 显示输出列的头部分。它和 -O 标志同时使用时，-O 标志会覆盖 -H 标志。
-l Name 指定检测名称。
-O 显示由冒号分隔的所有属性名称，在第二行显示由冒号分隔的对应的属性值。也指定了 -E 标志时，属性值是当前值，指定了 -D 标志时，属性值是缺省值。此标志不能与 -a 标志一起使用。

文件

/usr/sbin/shconf 包含 shconf 命令。
