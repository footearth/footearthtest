cthactrl 命令

用途

控制群集内的子系统。

语法

cthactrl -i <init_opt> | -s | -k | -b | -r | -d | -z | -h

描述

cthactrl 命令建立和控制群集子系统信息并管理拓扑结构服务和组服务。

标志

-i <init_opt>
初始化组服务和拓扑结构子系统，其中 <init_opt> 可以指定为：
-c <cluster_name>
指定群集名。
-n <nodenum>
指定节点号。
-e <environ>
指定包含群集访问模块的子目录。
[-p <portspec>]
指定组服务和拓扑结构服务的 UDP 端口号。

例如：

cthactrl -i -c filesys -n 1 -e filesys -p "cthats=12347,cthags=12348"
-s
启动组服务和拓扑结构服务子系统。
-k
停止组服务和拓扑结构服务子系统。
-b
重新构建组服务和拓扑结构服务子系统配置（例如，machines.lst）。
-r
刷新组服务和拓扑结构服务子系统。
-d
删除组服务和拓扑结构服务子系统。
-z
卸载组服务和拓扑结构服务子系统。
-h
将命令用法语句写至标准输出。

安全性

要运行该命令，必须拥有 root 权限。

退出状态

0
成功完成。
non-zero
发生故障。

限制

该命令仅适用于 cthags 和 cthats 子系统。

标准输出

当指定了 -h 标志，此命令的用法语句写至标准输出。

示例

  1. 要初始化本地节点作为 filesys1 群集的一部分，并指定 12347 作为 cthats 的 UDP 端口号和指定 12348 作为 cthags 的 UDP 端口号，输入： 
    cthactrl -i -c filesys1 -n 1 -p "cthats=12347,cthags=12348" -e filesys1
  2. 要启动组服务和拓扑结构服务子系统（cthags 和 cthats），输入： 
    cthactrl -s
  3. 要停止组服务和拓扑结构服务子系统（cthags 和 cthats），输入： 
    cthactrl -k

位置

/usr/sbin/rsct/bin/cthactrl

相关信息
