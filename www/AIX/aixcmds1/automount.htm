automount 守护进程

用途

安装自动安装点。

语法

/usr/sbin/automount [ -m ] [  -v ] [  -t Duration ] [  -i Interval ]

描述

automount 命令用作 AutoFS 的一个管理工具。它安装 AutoFS 安装点并使 automount 映射和每个安装点相关联。AutoFS 文件系统监视程序尝试访问它内部的目录并通知 automountd 守护进程。此守护进程使用映射来定位一个文件系统，然后将此系统安装在 AutoFS 文件系统中的引用点上。

如果不在适当的间隔（缺省为五分钟）内访问文件系统，则 automountd 守护进程卸装该文件系统。

包含命令行映射信息的文件名是 /tmp/autofs_cmdline。

如果 automountd 守护进程还没有启动，则 automount 命令尝试通过 SRC 启动它。

映射

映射的更新对用户而言是透明的，因为“名称到位置”的绑定是动态的。此进程消除了为包含对文件进行的硬编码引用的程序预先安装共享文件系统的需要。

有关格式化映射条目、多次安装、特殊映射和auto_master/auto.master NIS 配置映射文件的更多信息，请参阅《AIX 5L V5.2 系统管理指南：通信与网络》中的『如何管理 NIS 自动安装映射』。

-MountOptions 参数是一个安装选项的列表。此列表前面有一个 -（减号），且每个选项间由逗号隔开。如果提供此参数，这些选项通常成为映射中所有条目的缺省安装选项。

注：

  1. 映射中提供的安装选项覆盖 -MountOptions 参数。
  2. automount 守护进程不识别 mount 命令的 bg 后台安装选项。

automount 守护进程是单线程的。任何被慢速的或无响应的 NFS 服务器延迟的请求将在初始请求完成前延迟所有后续的自动安装请求。读取 /etc/mtab 文件并将接触驻留在自动安装点下的文件的程序将把更多的条目引入文件。自动安装的文件系统用 mount 命令的设置为与 ignore 相等的 -t 类型选项来安装。这些文件系统不出现在 mount 命令或 df 命令的输出中。

环境变量

仅特定于 automount 守护进程的环境变量可用于 automount 映射。当守护进程遇到 automount 变量，则环境扩展来容纳新变量。环境变量仅对于自动安装程序的环境才有效，对于操作系统环境是无效的。

通过将变量名称括在 { } (花括号) 中来防止引用带后缀的字符。

注：操作系统不支持一些 NFS 服务器支持的安装选项：grpid、noauto、remount、quota、noquota、posix、nocto 和 noac。缺省情况下，这个版本的 automount 守护进程忽略这些列出的选项。要逆转此缺省值的效果，请使用 AUTOMOUNT_BAD_OPTS shell 环境变量。

配置

automount 守护进程通常参考 auto.master NIS 配置映射来获得初始 目录至映射名称（Directory-to-MapName）对的列表，除了那些在命令行给出的还为它们进行自动安装。如果有完全相同的情况，则命令行参数取在先的那个。

注：此映射包含 automount 守护进程参数。automount 守护进程不会在本地主机上查找 auto.master 文件。

有关配置 auto.master NIS 映射文件的更多信息，请参阅《AIX 5L V5.2 系统管理指南：通信与网络》中的『如何管理 NIS 自动安装映射』。

标志

-i Interval 指定 Autofs 目录（自动安装的目录）存在的间隔（以秒为单位）。
-m 指定不要搜索 NIS 的自动安装映射。
-t Duration 指定 Autofs 卸装线程在它再次开始工作之前处于睡眠状态的持续时间（以秒为单位）。
-v 在标准输出上显示详细状态和警告消息。对于两种实现，此标志都受到支持。

文件

/tmp_mount 包含在其下动态安装文件系统的目录。
auto_master 或 auto.master 包含 automount 守护进程的 NIS 配置映射。
