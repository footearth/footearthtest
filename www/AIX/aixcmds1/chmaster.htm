chmaster 命令

用途

chmaster 命令执行 ypinit 命令，并重新启动 NIS 守护进程来更改主控服务器。

语法

/usr/etc/yp/chmaster [ -s HostName [ , HostName ... ] ] [ -O | -o ] [ -E | -e ] [ -P | -p ] [ -U | -u ] [ -C | -c ] [ -I | -B | -N ]

描述

chmaster 命令调用 ypinit 命令来为当前域更新 NIS 映射，假定当前设置了系统的域名。ypinit 命令成功完成后，chmaster 命令为 ypserv 命令、yppasswdd 命令、ypupdated 命令以及 ypbind 命令注释或取消注释 /etc/rc.nfs 文件中的项。

可以使用 基于 Web 的系统管理器（wsm）中的 网络 应用程序来更改网络特征。还可以使用系统管理界面程序（SMIT）smit chmaster 快速路径来运行此命令。

标志

-B 更新 /etc/rc.nfs 文件以启动适当的守护进程，调用 ypinit 命令并启动守护进程。
-C 将 ypbind 守护进程与 ypserv 守护进程一起启动。此标志是缺省值。
-c 禁止 ypbind 守护进程的启动。
-E 如果遇到错误，从 ypinit 命令和 chmaster 命令退出。此标志是缺省值。
-e 如果遇到错误，禁止从 ypinit 命令和 chmaster 命令退出。
-I 指示 chmaster 命令更改 /etc/rc.nfs 文件以在下一次系统重新启动时启动适当的守护进程。当调用此命令时，会执行 ypinit 命令。
-N 调用 ypinit 命令并启动适当的守护进程。不对 /etc/rc.nfs 文件进行更改。
-O 覆盖此域的现有映射。
-o 防止 NIS 映射的覆盖。此标志是缺省值。
-P 将 yppasswdd 守护进程与 ypserv 守护进程一起启动。
-p 禁止 yppasswdd 守护进程的启动。 此标志是缺省值。
-s HostName [, HostName ] 为此主控服务器的从服务器指定从主机名。 chmaster 命令自动将当前主机添加到此列表。
-U 将 ypupdated 守护进程与 ypserv 守护进程一起启动。
-u 禁止 ypupdated 守护进程的启动。 此标志是缺省值。

示例

要调用 ypinit 命令来为当前域重新构建 NIS 映射，请输入：

chmaster -s chopin -O -p -u -B

在此示例中，chmaster 命令覆盖现有的映射，且 yppasswdd 和 ypupdated 守护进程不启动。将主机名 chopin 指定为一个从服务器。

文件

/etc/rc.nfs 包含 NFS 和 NIS 守护进程的启动脚本。
/var/yp/domainname 包含 NIS 域的 NIS 映射。
