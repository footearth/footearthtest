cthatsctrl 命令

用途

控制拓扑服务子系统。

语法

cthatsctrl { -a [ -p port-number ] | -s | -k | -d | -b | -t | -o | -r | -h }

描述

cthatsctrl 控制命令控制拓扑服务子系统的操作。该子系统处在系统资源控制器（SRC）的控制之下，并属于一个称作 cthats 的子系统组。与每个子系统关联的是一个守护进程及一条用来配置和启动守护进程的命令。

群集的每个节点上都运行着拓扑服务子系统的实例。

添加子系统

当指定了 -a 标志时，此控制命令将使用 mkssys 命令添加拓扑服务子系统到 SRC。控制命令：

  1. 确保 cthats 子系统已停止。
  2. 从群集数据获取端口号，确保端口号在 /etc/services 文件中设置。

    输入在 /etc/services 文件中的服务名称为 cthats。

  3. 从 SRC 除去 cthats 子系统（在它仍然在其中的情况下）。
  4. 添加 cthats 子系统至 SRC。

启动子系统

当指定了 -s 标志时，此控制命令将使用 startsrc 命令开始启动拓扑服务子系统 cthats。

停止子系统

当指定了 -k 标志时，此控制命令将使用 stoptsrc 命令停止拓扑服务子系统 cthats。

删除子系统

当指定了 -d 标志时，此控制命令将使用 rmssys 命令从 SRC 删除拓扑服务子系统。控制命令：

  1. 确保 cthats 子系统已停止。
  2. 使用 rmssys 命令从 SRC 中除去 cthats 子系统。
  3. 从 /etc/services 文件中除去 cthats 端口号。

重新建立配置

当指定了 -b 标志时，该控制命令从群集数据中读取配置信息并为拓扑服务守护进程建立一个配置文件 machines.lst。

打开跟踪

当指定了 -t 标志时，此控制命令将通过使用 traceson 命令为拓扑服务守护进程打开跟踪。

关闭跟踪

当指定了 -o 标志时，此控制命令将通过使用 tracesoff 命令为拓扑服务守护进程关闭跟踪（将其恢复为缺省级别）。

刷新子系统

当指定了 -r 标志时，此控制命令将通过使用 refresh 命令刷新子系统。-r 标志向守护进程发送读取重建信息的信号。

标志

-a [-p port-number]
添加子系统。
-s
启动子系统。
-k
停止子系统。
-d
删除子系统。
-t
打开子系统跟踪。
-o
关闭子系统跟踪。
-b
从群集数据中的配置信息重建拓扑服务配置文件。
-r
刷新子系统。
-h
将命令用法语句写至标准输出。

安全性

要运行该命令，必须拥有 root 权限。

退出状态

0
指明命令成功完成。
一个非零值
指明发生错误。

限制

该命令仅在对等域中有效。

仅在“IBM 支持中心”的指导下使用该命令。

标准输出

当指定了 -h 标志，此命令的用法语句写至标准输出。

标准错误

该命令写任何错误消息至标准错误。

示例

  1. 要添加拓扑服务子系统至 SRC，输入： 
    cthatsctrl -a
  2. 要启动拓扑服务子系统，输入： 
    cthatsctrl -s
  3. 要停止拓扑服务子系统，输入： 
    cthatsctrl -k
  4. 要从 SRC 中删除拓扑服务子系统，输入： 
    cthatsctrl -d
  5. 要打开拓扑服务守护进程的跟踪，输入： 
    cthatsctrl -t
  6. 要关闭拓扑服务守护进程的跟踪，输入： 
    cthatsctrl -o
  7. 从群集数据中的配置信息重建拓扑服务配置文件，输入： 
    cthatsctrl -b
  8. 要向群集中的所有拓扑服务守护进程发送读取新建配置文件的信号，输入： 
    cthatsctrl -r
  9. 要写使用信息至标准输出，输入： 
    cthatsctrl -h

位置

/usr/sbin/rsct/bin/cthatsctrl
包含 cthatsctrl 命令
