chlvcopy 命令

用途

将镜像副本标记为分割镜像或取消将其标记为分割镜像。

语法

chlvcopy [  -f ] {  -B  [  -s ] } | {  -b  [  -c copy ] [  -f ] [  -P ] [  -l newlvname ] [  -w ] } LV name

描述

注:
  1. 要使用此命令，必须具有 root 用户权限或是系统组的成员。
  2. 如果通过使用 -P 标志使用持久性，或者通过使用 -l 标志创建子备份逻辑卷设备来使用持久性，则它将使卷组只在 AIX 4.3.2 或更新版本上可用。即使在除去父逻辑卷和子备份逻辑卷的分割镜像副本指定后，这一点仍然适用。
  3. 要使 chlvcopy 在并行卷组环境中成功运行，所有并行节点必须为 AIX 4.3.2 或更新版本。
  4. 如果逻辑卷在具有快照卷组的卷组或快照卷组中，则不允许 chlvcopy 命令。
  5. 应将 chfs 用于在文件系统位于要复制的逻辑卷上时创建分割镜像副本。

在 chlvcopy 可以将镜像副本标记为分割镜像之前，逻辑卷的所有分区都必须刷新。 只有一个副本可以指定为联机分割镜像副本。

虽然 chlvcopy 命令可以标记打开的逻辑卷（包括包含安装的文件系统的逻辑卷）上的联机分割镜像副本，但不推荐这样做，除非应用程序在将副本标记为分割镜像时处于已知状态。在运行 chlvcopy 命令时，分割镜像副本内部一致，但是如果多个进程同时访问逻辑卷并且应用程序未处于已知状态，则逻辑卷和分割镜像副本之间的一致性丢失。当标记打开的逻辑卷时，数据可能丢失或毁坏。在标记联机分割镜像副本之前应关闭逻辑卷以避免潜在的数据毁坏窗口。

如果没有设置持久性标志来防止备份数据的丢失，则卷组应该设置为不自动联机，并且 -n 标记应与 varyonvg 一起使用以防止重新同步旧文件分区。如果设置了持久性标志（-P），则以下情况适用：如果在联机分割镜像副本存在（或多个存在）时发生崩溃，则系统重新引导时保留副本的存在。

标志

-b 将镜像副本标记为分割镜像副本。
-c copy 要标记为分割镜像副本的镜像副本。副本的允许值为 1、2 或 3。如果没有指定此选项，则副本的缺省值为逻辑卷的最后一个镜像副本。
-B 取消将镜像标记为分割镜像副本。如果使用 -l 选项创建一个副本，则它还将试图除去子备份逻辑卷。
-f 即使有旧文件分区，也强制分割镜像副本。如果与 -B 选项一起使用，则当使用 -l 选项创建一个副本时，将使用 force 选项除去子备份逻辑卷。
-l newlvname 备份逻辑卷的新名称。指定 -l 标记还会设置持久性选项，允许应用程序通过 newlvname 访问分割镜像副本。
-P 保留有关通过重新引导的联机分割镜像副本的存在的信息，还允许其它节点（在并行方式环境下）知道联机分割镜像副本的存在。
-s 为逻辑卷启动后台 syncvg。
-w 允许分割镜像副本可写（缺省值为将分割镜像副本创建为“只读”）。
LV name 要对其操作的逻辑卷。
