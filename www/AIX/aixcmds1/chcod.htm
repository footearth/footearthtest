chcod 命令

用途

管理按需升级容量。

语法

chcod [ -r ResourceType -n NbrResources ] [-c CustomerInfo ] [ -m MailAddr ] [-h ]

描述

chcod 命令管理按需升级容量或CUoD。CUoD启用比初始授权更多的 ResourceTypes 授权，如处理器。如果附加资源可用并且系统支持 ResourceType 的CUoD，则可以启用它们。一次只可以管理一种 ResourceType。ResourceTypes 数量的更改要到下次系统引导之后才生效。CUoD管理也包括显示具有CUoD支持的 ResourceType 的当前数量、监控系统上的 ResourceType 的数量并适当地发送通知。 每月会发送一次通知，每当 NbrResources 更改时，也会进行通知。

通知采用错误日志记录的形式，也可以采用发送电子邮件的形式。 每当指定的 ResourceType 更改了以及基于每个月份，都会在系统错误日志中加入条目。CustomerInfo 文本被包括在错误日志中。如果使用 MailAddr 指定了电子邮件地址，则通知也会通过电子邮件消息发送至 MailAddr。CustomerInfo 文本被包括在消息文本中。如果指定 CustomerInfo 和 MailAddr，则可以通过错误日志和电子邮件进行通知。

如果未指定日志，则 chcod 显示 CustomerInfo、MailAddr、系统模块名称和序列号的当前值以及任何具有CUoD支持的ResourceType 的 NbrResources 的当前值。

注:
从 IBM p650 和更高模型（所有 POWER4 系统）开始，CUoD 在硬件管理控制台（HMC）上管理。

标志

-c CustomerInfo 指定文本字符串包括在错误日志中。该字符串也包括在任何已发送电子邮件消息的正文中。CustomerInfo 不能超过 255 个字符。字符串中不能包括空格。指定 CustomerInfo 后，后续的 chcod 使用不必指定 -c 标志，但必须具有更改该标志的选项。CustomerInfo 可以由字母数字字符以及任何 .（句点）、,（逗号）、-（连字符）、(（开括号）或 )（闭括号）组成。
-h 显示使用消息。
-m MailAddr 指定电子邮件将发送至的电子邮件地址。MailAddr 不可以超过 255 个字符。 如果通过指定 ""（空白字符串）重新设置 MailAddr，则只有错误日志会监控具有 CUoD 支持的资源。如果想要将通知发送到该电子邮件地址，必须已在系统上配置了电子邮件。
-n NbrResources 指定系统上要授权的 ResourceTypes 的数量。必须是零或大于零。如果是零，则对于指定的 ResourceType，CUoD禁用。如果指定 -n，则也必须指定 -r。
-r ResourceType 将 ResourceType，例如用于处理器的 proc，指定为在系统上启用并受监视。该系统必须为 ResourceType 支持 CUoD。如果指定 -r，则也必须指定 -n。

示例

  1. 要为处理器启动CUoD，请输入： 
    chcod -r proc -n 10 -m"someone@ibm.location.com" -c"Jane_Doe-Customer_Number_999999-(111)111-1111"
  2. 要更改 CustomerInfo，请输入： 
    chcod -c"Jane_Doe-Customer_Number_999999-(222)222-2222"
  3. 要停止通知的电子邮件格式，请输入： 
    chcod -m""
  4. 要查看具有CUoD支持的资源的当前值，请输入： 
    chcod

    会显示同以下内容相似的消息：

    当前 CustomerInfo = Jane_Doe-Customer_Number_999999-(222)222-2222
当前 MailAddr = someone@ibm.location.com
当前型号与序列号 = IBM,7043-150 000974934C00
当前已授权的处理器数 = 系统上已安装的 12 个中的 10 个
