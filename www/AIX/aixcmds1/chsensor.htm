chsensor 命令

用途

更改资源监控和控制（RMC）传感器的属性。

语法

chsensor [-i seconds] [-a | -n host1[,host2...]] [-h] [ -v | -V ] name [Name=new_name]

描述

chsensor 命令更改传感器的名称或更新传感器值的时间间隔。传感器通过 name 标识。如果 new_name 是包含空格或特殊字符的字符串，则必须将其用引号引起来。

chsensor 命令可在任何节点上运行。如果希望 chsensor 在域中的所有节点上运行，请使用 -a 标志。如果要使 chsensor 在域中的某个子网的节点上运行，请使用 -n 标志。

传感器是一种拥有可监视的属性的 RMC 资源。属性的值由在指定的时间间隔时更新值的传感器资源管理器设置（更新操作使用 mksensor 命令指定的命令或脚本进行）。传感器值也可使用 refsensor 命令更新。

标志

-a
更改域中所有节点上匹配指定名称的传感器。 CT_MANAGEMENT_SCOPE 环境变量确定群集作用域。如果未设置 CT_MANAGEMENT_SCOPE，请首先选择管理域作用域（如果存在），再选择对等域作用域（如果存在），然后选择本地作用域，直到作用域对命令有效为止。将运行该命令一次以找到第一个有效域。例如，如果同时存在管理域和对等域，没有设置 CT_MANAGEMENT_SCOPE 的 chsensor -a 将在管理域中运行。在这种情况下，要在对等域中运行，请将 CT_MANAGEMENT_SCOPE 设置为 2。
-i seconds
指定运行 sensor 来更新传感器属性的值的时间间隔。seconds 为整数值，必须大于或等于 10。仅当传感器资源受监视时才在指定的时间间隔运行该 sensor 命令。如果时间间隔设置为 0，将不自动运行 sensor 命令。使用 refsensor 命令独立于时间间隔更新。
-n host1[,host2...]
指定应该在其上更改传感器的节点。缺省情况下，在本地节点上更改传感器。该标志只适用于管理域或对等域中。
-h
将命令用法语句写至标准输出。
-v | -V
将命令的详细消息写至标准输出。

参数

name
指定要更改的传感器名称。
Name=new_name
指定为传感器资源提供的新名称。

安全性

用户需要 IBM.Sensor 资源类的写许可权以运行 chsensor。许可权在联系系统的访问控制表（ACL）文件中指定。

退出状态

0
命令已成功运行。
1
输入了不正确的标志和参数的组合。
6
未找到传感器资源。
n
基于 RMC 子系统可能返回的其它错误。

环境变量

CT_CONTACT
当 CT_CONTACT 环境变量设置为主机名或 IP 地址时，该命令联系指定的主机上的资源监控和控制（RMC）守护进程。如果未设置此变量，则该命令联系运行命令的本地主机上的 RMC 守护进程。该命令显示或修改的资源类或资源位于连接建立的系统上。
CT_MANAGEMENT_SCOPE
确定用于同 RMC 守护进程进行会话的管理作用域，该守护进程用来监视和控制资源和资源类。 该管理作用域确定了能被监视和控制的资源和资源类所在的可能目标节点集。

有效值为：

0
指定本地作用域。
1
指定本地作用域。
2
指定对等域作用域。
3
指定管理域作用域。

如果未设置此环境变量，则使用本地作用域。

示例

  1. 要将 SensorA 传感器的 Name 更改为 Sensor1A，请输入： 
    chsensor SensorA Name=Sensor1A
  2. 要将 SensorA 传感器的更新时间间隔更改为 10，请输入： 
    chsensor -i 10 SensorA

位置

/usr/sbin/rsct/bin/chsensor
包含 chsensor 命令
