chslave 命令

用途

重新执行 ypinit 命令以从主控服务器检索映射并重新启动 ypserv 守护进程以更改从属服务器。

语法

/usr/etc/yp/chslave [ -C | -c] [ -O | -o ] [ -I | -B | -N ] Master

描述

chslave 命令重新调用 ypinit 命令以从命令行上指定的主控服务器检索映射。在 ypinit 命令成功完成后，重新启动 ypserv 守护进程。Master 参数指定主控服务器的主机名。指定的主控服务器可以是当前正在使用的主控服务器或已配置并正在运行的新主控服务器。

可以使用“基于 Web 的系统管理器”（wsm）中的“网络”应用程序更改网络特征。也可以使用系统管理界面程序（SMIT）smit chslave 快速路径运行此命令。

标志

-B 调用 ypinit 命令并启动 ypserv 守护进程。如果 ypserv 守护进程已在运行，则此标志将导致 ypinit 命令杀死守护进程，然后重新启动它。这个标志是缺省的。
-C 使用 -n 标志调用 ypinit 命令。chslave 命令发生错误时继续运行。 这个标志是缺省的。
-c 当错误发生时停止执行。
-I 立即执行 ypinit 命令，但不启动或重新启动 ypserv 守护进程。
-O 覆盖域中存在的任何映射。
-o 防止覆盖域中存在的映射。这个标志是缺省的。
-N 调用 ypinit 命令并重新启动 ypserv 守护进程。

示例

要从名为 host91 的主控服务器检索映射，请输入：

chslave -O -B host91

这将为当前域覆盖任何现有映射。

文件

/etc/rc.nfs 包含 NFS 和 NIS 守护进程的启动脚本。
/var/yp/domainname 包含 NIS 域的 NIS 映射。
