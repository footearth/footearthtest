xwud 命令

用途

检索和显示增强的 X-Windows 的转储图像。

语法

xwud [ -in FileName ] [ -noclick ] [ -geometry Geometry ] [ -display Display ] [ -new ] [  -std MapType ] [ -raw ] [ -vis visual_type | visual_id ] [ -help ] [ -rv ] [ -plane Number ] [ -fg Color ] [ -bg Color ]

描述

xwud 命令检索增强的 X-Windows 窗口的转储图像。它通过在一个视窗中显示由 xwd 命令 先前产生的特殊格式的转储文件来实现上述功能。转储文件格式由 XWDFile.h 文件决定。

可以用标志指定彩色显示器、视窗大小及位置、输入字段和可视类或标识。也可以选择要显示的图像单一层面。

标志

-bg Color 指定在显示一幅位图图像（或图像的单一层面）时图像中显示的 0（零）位的颜色。
-display Display 指定要连接的服务器；请参阅 X 命令。
-fg Color 指定在显示一幅位图图像（或图像的单一层面）时图像中显示的 1 位的颜色。
-geometry Geometry 指定视窗的大小和位置。典型用法是只要指定位置，让大小缺省设置为图像 的实际大小。
-help 打印允许选项的简短描述。
-in FileName 在命令行指定输入文件。如果没有指定输入文件，就采用标准输入。
-new 为显示图像创建一个新的颜色映射。如果图像特征匹配显示器的特征， 这个标志可以在屏幕上更快地显示图像，但这是以使用新的颜色映射为代价的（在很多终端上这会导致其它视窗也显示为彩色）。
-noclick 在单击视窗中的一个按钮时阻止应用程序结束。通过输入字符 q 或 Q，或是用 Ctrl-C 按键顺序来结束应用程序。
-plane Number 选择要显示的图像单一层面。平面用数字标识，0（零）是最低有效位。用该标志决定哪个 层面传递到 xpr 命令打印。
-raw 用当前屏幕上存在的任何颜色显示转储图像。这个标志在回存图像到图像原来所在的相同 屏幕，而该源视窗还在屏幕上时很有用。这样可以更快地在屏幕上获取图像。
-rv 如果位图图像（或图像的单一层面）显示时交换前景色和背景色。这个标志在显示区分像素值 为 0 和 1（从其在显示器上的显示逆向获得）的颜色的位图图像时很有用。
-std MapType 使用指定的标准颜色映射显示图像。通过将类型转换为大写字母，并加 前缀 RGB_ 和添加后缀 _MAP，可以获取映射类型。典型 的映射类型为 best、default 和 gray。请参阅 /usr/lpp/X11/Xamples/clients/xstdcmap 有关创建标准颜色映射的信息。
-vis visual_type | visual_id 指定特定的可视类型或可视标识。缺省值选取最好 的一个，或者您指定 default，这和根视窗的颜色映像图的类相同。

也可以指定一个特定的类：StaticGray、GrayScale、StaticColor、PseudoColor、DirectColor、TrueColor。指定 Match 使用和源图像一样的类。

指定一个十六进制的数（以 0x 为前缀）或是一个十进制数作为确切的可视标识（对服务器是特定的）。该字符串不区分大小写。

环境变量

DISPLAY 获取缺省显示器。

示例

要在转储视窗检索一个特定的文件，输入：

xwud -in FileName
