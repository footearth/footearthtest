xrx 命令

用途

RX 帮助器程序。

语法

xrx [ -ToolKitOption ...] filename

描述

帮助器程序可以与任何 web 浏览器一起使用，用来解释 RX MIME 类型格式的文档并启动远程应用程序。

xrx 读入用文件名指定的 RX 文档，从那里它获取应用程序要使用的服务列表。根据这个信息，xrx 设置不同的请求服务，包括 如果 X 服务器支持 SECURITY 扩展则创建授权密钥。然后它通过 一个相关的 CGI 脚本的 HTTP GET 请求，将相关数据，例如 X 显示器名称，传递到应用程序。Web 服务器接着执行 CGI 脚本以启动应用程序。客户机在连接到 X 服务器上的 Web 服务器主机上运行。

安装

需要配置 web 浏览器以使用xrx 打开 RX 文档。通常 $HOME/.mailcap 中有下列行就足够了：

application/x-rx; xrx %s

不过，可能需要参看 web 浏览器文档中关于配置帮助器应用程序的确切的指导信息。

一旦正确地完成配置，无论何时检索任何 MIME 类型 application/x-rx 的文档，浏览器就将激活帮助器程序。

标志

-ToolKitOption 这个xrx 帮助器接受所有标准 X Toolkit 命令行选项，例如：
xrm resourcestring
本选项指定要使用的资源字符串。在命令行也许有几个该选项的实例。

资源

xrx 程序的应用程序类名是 Xrx，它可以接受并 解释以下应用程序资源名和类：

xrxHasFirewallProxy（类 XrxHasFirewallProxy）
指定 X 服务器防火墙是否正在运行且是否应当使用。缺省值为“False”。
xrxInternalWebServers（类 XrxInternalWebServers）
不应使用 X 服务器防火墙代理的 Web 服务器（只有当 xrxHasFirewallProxy 是“True”有关系）。它的值是逗号分隔的掩码／值对列表，它们用于根据其地址来过滤内部 Web 服务器。“掩码”部分指定要考虑地址的哪些段而“值”部分则指定结果应该匹配的内容。例如，以下列表：
255.255.255.0/198.112.45.0,
255.255.255.0/198.112.46.0
匹配地址集：198.112.45.* 和 198.112.46.*。更加精确地说，测试就是(address &mask) == value。
xrxFastWebServers（类 XrxFastWebServers）
不应使用 LBX 的 Web 服务器。资源值是一个地址掩码／值对列表，这在以上已经描述。
xrxTrustedWebServers（类 XrxTrustedWebServers）
远程应用程序应当作为可信的客户机运行的 Web 服务器。缺省值是作为不可信的客户机运行远程应用程序。资源值是一个地址掩码／值对列表，这在以上已经描述。

环境

xrx 帮助器使用标准 X 环境变量，比如 DISPLAY，用以获取缺省的 X 服务器 主机和显示器号码。如果 RX 文档请求 X-UI-LBX 服务而缺省 X 服务器没有通告 LBX 扩展，xrx 将寻找环境变量 XREALDISPLAY 来获取 X 服务器的另一地址并在该处寻找 LBX 扩展。当通过 lbxproxy 运行浏览器时，如果 希望远程应用程序能够通过因特网使用 LBX 服务，则需要将 XREALDISPLAY 设置为服务器的确切地址。

如果 RX 文档请求 XPRINT 服务，xrx 将寻找变量 XPRINTER、PDPRINTER、LPDEST、PRINTER 和 XPSERVERLIST 来获取要使用的打印机名和 X 打印服务器的地址。注意：尽管该组变量允许指定多个服务器和打印机，但只使用第一个服务器和打印机对。最后，如果正在使用防火墙代理，xrx 将寻找 PROXY_MANAGER 来获取代理管理器的地址（请参阅proxymngr）。当没有指定时，它会使用 :6500 作为缺省值。
