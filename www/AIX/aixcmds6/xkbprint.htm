xkbprint 命令

用途

打印 XKB 键盘描述。

语法

xkbprint [ -? | -help ] [ -color ] [ -dflts ] [ -diffs ] [ -eps ] [ -fit ] [ -full ] [ -grid Resolution ] [ -if FontName ] [ -label Type ] [ -lc Locale ] [ -level1 ] [ -level2 ] [ -lg Group ] [ -ll Level ] [ -mono ] [ -n Number ] [ -nkg Number ] [ -npk Number ] [ -o File ] [ -R Directory [ -pict Which] ] Source [ OutputFile ]

描述

xkbprint 命令生成一个由 Source 指定的 XKB 键盘描述的可打印或封装的 PostScript 描述。Source 可以是任何编译过的键映射（.xkm）文件，包括一个几何描述或 X 显示规范。如果指定了一个 OutputFile，那么 xkbprint 就会向它写数据。否则，xkbprint 将创建输出文件，根据请求的格式用 .ps 或 .eps 代替源文件的扩展。如果源为非本地 X 显示（例如 :0），xkbprint 就会将适合的前缀附加在显示规范上，用 -（横杠）代替冒号。对于本地显示，xkprint 将使用 server-n，其中 n 为显示号。

标志

-? | -help 打印用法消息。
-color 使用几何文件中指定的颜色进行打印；缺省情况下，xkbprint 打印一个键盘的黑白图像。
-dflts 试图计算任何丢失的组件（如密钥）的缺省名称。
-diffs 仅在符号明确的绑定时显示它们。
-eps 生成一个封装的 PostScript 文件。
-fit 安排页面上的键盘映像，这是缺省值。
-full 全尺寸打印键盘。
-grid Resolution 用键盘上的 Resolutionmm 分辨率打印网格。
-if FontName 如果不指定输出文件，就指定内部 PostScript 类型 1 字体来转储到指定的输出文件或 fontName.pfa 上。如果转储的是内部字体，那么不打印任何键盘描述。
-label Type 指定要根据密钥打印的标签。有效的类型为：
  • none
  • name
  • code
  • symbols
-lc Locale 指定 KeySyms 应该解析的语言环境。
-level1 生成级别 1 的 PostScript。
-level2 生成级别 2 的 PostScript。
-lg Group 从 Group 开始，打印键盘组中的符号。
-ll Level 从换档级别 Level 开始打印符号。
-mono 生成键盘的黑白图像，这是缺省值。
-n Number 打印副本的 Number。
-nkg Number 打印 Number 键盘组中的符号。
-npk Number 指定每页要打印的键盘图像的 Number。对于 EPS 文件，它指定要打印的键盘图像的总数。
-o File 将输出写到 File。
-R Directory 使用 Directory 作为根目录，所有路径名都根据 Directory 来解释。
-pict Which 控制统计图表（而不是可用的 keysym 名称）的使用。Which 的有效值为：
  • all
  • none
  • common (default).
-synch 强制同步 X 请求。
-w Level 设置警告级别。
  • 0 就是没有警告
  • 10 就是全都警告
