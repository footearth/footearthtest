hagsd 守护进程

用途

观察由资源监视器更新的资源变量实例，产生并报道客户程序事件.

语法

hagsd [-a] [-s] [-k] [-d] [-c] [-u] [-t] [-o] [-r] [-h] daemon_name

描述

hagsd 守护进程是组服务子系统的一部分, 它提供了一个通用功能来协调和监视在群集节点上运行的应用程序状态的变化. . 该守护进程提供子系统的大多数服务。daemon_name 指定守护进程使用的名称来命名日志文件并在 AIX 错误日志中识别其消息。

一个hagsd 守护进程实例在每个群集节点上执行. hagsd 守护进程受系统资源控制器(SCR)的控制.

由于守护进程在 SRC 控制下,最好不要从命令行直接启动它. 在正常情况下守护进程由grpsvcsctrl 命令来调用,该命令由群集启动进程来依次调用. 如果一定要直接启动或停止守护进程，则使用startsrc 或 stopsrc命令.

标志

-a
添加子系统.
-s
启动子系统.
-k
停止子系统.
-d
删除子系统.
-c
清除子系统，也就是说, 从所有系统分区中删除它们.
-u
解除子系统在所有系统分区中的配置.
-t
打开子系统跟踪.
-o
关闭子系统跟踪.
-r
刷新子系统.
-h
显示用法信息.

参数

deamon_name
指定守护进程用以命名日志文件和识别在 AIX 错误日志中的消息的名称。

安全性

您必须有root权限来运行脚本程序.

退出状态

0
指示命令已成功完成.
1
显示有错误产生.

限制

此命令只在 PSSP 环境下有效.

标准输出

当-h 标志 被指定时, 该命令的用法说明写到标准输出中去.

标准错误

必要时此命令把错误信息写到标准错误中。

示例

  1. 在当前系统分区中，将组服务子系统添加到 SRC 上, 设置 SP_NAME 环境变量为正确的系统分区名，键入: 
    hagsctrl -a
  2. 在当前系统分区中，启动组服务子系统, 设置 SP_NAME 环境变量到正确的系统分区名，键入: 
    hagsctrl -s
  3. 在当前系统分区中，停止组服务子系统, 设置 SP_NAME 环境变量为正确的系统分区名，键入: 
    hagsctrl -k
  4. 在当前的系统分区中，从 SRC 上删除组服务子系统, 设置 SP_NAME 环境变量为正确的系统分区名，键入: 
    hagsctrl -d
  5. 在所有系统分区中，清除组服务子系统，键入: 
    hagsctrl -c
  6. 在控制工作站上，从所有系统分区，解除组服务子系统的配置，键入: 
    hagsctrl -u
  7. 在当前系统分区中，打开服务组 daemon 的跟踪程序, 设置 SP_NAME 环境变量为正确的系统分区名，键入: 
    hagsctrl -t
  8. 在当前系统分区中，关闭服务组 daemon 的跟踪程序, 设置环境变量 SP_NAME 为正确的系统分区名，键入: 
    hagsctrl -o

位置

/usr/sbin/rsct/bin/hagsd
包含hagsd 守护进程

文件

/var/ha/log/hags_nodenum_instnum.syspar_name
包含节点上 hagsd 守护进程的日志
/var/ha/log/hags.syspar_name_nodenum_instnum.syspar_name
包含控制工作站上的每一 hagsd 守护进程的日志.

文件名包含下列变量:

  • nodenum 是有守护进程运行的节点号。
  • instnum 守护进程的实例数目。
  • syspar_name 是有守护进程运行的系统分区的名称。
