haemd_HACMP 命令

用途

事件管理器守护进程的启动程序。

语法

haemd_HACMP [ -d trace_arg ]

描述

haemd_HACMP 程序是 haemd 守护进程的启动程序。当 emsvcsctrl 命令在系统资源控制器（SRC）中配置事件管理子系统时，haemd_HACMP 被指定为启动程序。

该程序只能由 SRC 调用。要启动事件管理子系统，请使用 emsvcsctrl 命令。

-d 标志应该仅在 IBM 支持中心的指导下使用。可供使用的跟踪参数除 regs 和 dinsts 外，其余的与 haemtrcon 命令的相同。要使用此标志，必须使用带 -a 参数的 chssys 命令对 SRC 中的 emsvcs 子系统定义进行更改。随后必须停止该守护进程并重新启动它。

标志

-a
添加子系统。
-s
启动子系统。
-k
停止子系统。
-d
删除子系统。
-c
清理子系统，即从所有系统分区中将它们删除。
-u
从所有系统分区取消对子系统的配置。
-t
启用子系统跟踪功能。
-o
禁用子系统的跟踪功能。
-r
刷新子系统.
-h
显示用法信息.

参数

无。

安全性

您必须有 root 特权和对 SDR 的写权限以运行这个命令.

退出状态

0
指示命令已成功完成.
1
显示有错误产生.

限制

此命令仅在 HACMP 环境中有效。

标准输出

当命令执行成功时，写入下列信息消息:

读分区syspar_name的事件管理数据
 
CDB=new_EMCDB_file_name Version=EMCDB_version_string

标准错误

必要时此命令把错误信息写到标准错误中。

示例

  1. 向当前系统分区的 SRC 添加事件管理子系统，请将 SP_NAME 环境变量设置成相应系统分区的名称，然后输入： 
    haemctrl -a
  2. 启动当前系统分区中的事件管理子系统，请将 SP_NAME 环境变量设置成相应系统分区的名称，然后输入： 
    haemctrl -s
  3. 停止当前系统分区中的事件管理子系统，请将 SP_NAME 环境变量设置成相应系统分区的名称，然后输入： 
    haemctrl -k
  4. 从当前系统分区的 SRC 中删除事件管理子系统，请将 SP_NAME 环境变量设置成相应系统分区的名称，然后输入： 
    haemctrl -d
  5. 清理所有系统分区中的事件管理子系统，请输入： 
    haemctrl -c
  6. 在控制工作站上从所有系统分区中取消对事件管理子系统的配置，请输入： 
    haemctrl -u
  7. 启用当前系统分区中事件管理守护进程的跟踪功能，请将 SP_NAME 环境变量设置成相应系统分区的名称，然后输入： 
    haemctrl -t
  8. 关闭当前系统分区中事件管理守护进程的跟踪功能，请将 SP_NAME 环境变量设置成相应系统分区的名称，然后输入： 
    haemctrl -o

位置

/usr/sbin/rsct/bin/haemd_HACMP
haemd_HACMP 程序所在的位置

文件

/var/ha/log/em.default.cluster_name
包含名为 cluster_name 的群集上的 haemd 守护进程的缺省日志。
/var/ha/log/em.cp
包含在复制事件管理配置数据库时所产生的任何错误的日志。
/var/ha/log/em.trace.cluster_name
包含名为 cluster_name 的群集上的 haemd 守护进程的跟踪日志。
/var/ha/log/em.mkgroup
包含在创建 haemrm 组时所产生的任何错误的日志。
/var/ha/log/em.mkdir
包含在创建 /var/ha/lck/haem 和 /var/ha/soc/haem 目录时所产生的任何错误的日志。
