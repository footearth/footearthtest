fingerd 守护进程

用途

为 finger 命令提供服务器功能。

语法

注:
fingerd 守护进程通常被 inetd 守护进程启动。使用系统资源控制器（SRC）命令也能通过命令行控制它。

/usr/sbin/fingerd [ -s] [ -f]

描述

/usr/sbin/fingerd 守护进程是一个简单的协议，它给在几个网络站点里的 finger 命令提供接口。finger 命令返回一个关于当前系统或者一个用户的状态报告。fingerd 守护进程侦听在 /etc/services 文件和文件 /etc/inetd.conf 中列出的端口 79 处的传输控制协议（TCP）请求。

对个别的涉及到 fingerd 守护进程的安全性站点，作为缺省值，将不会转发任何的 finger 请求到其它的系统。如果它接受到一个 finger 转发的请求，fingerd 守护进程用 Finger 转发服务被禁止的信息来发到 finger 命令上。当使用-f 标志运行 fingerd 守护进程的时候，系统管理员可以有打开 finger 转发作为缺省值的选项。

fingerd 守护进程的改动可以通过使用系统管理界面程序（SMIT）或者 SRC 或者通过编辑 /etc/inetd.conf 文件或者 etc/services 文件来实现。不推荐在命令行输入 fingerd。当在 /etc/inetd.conf 文件未注释时，fingerd 守护进程缺省启动。

inetd 守护进程从 /etc/inetd.conf 文件和 /etc/services 文件中得到它的信息。

在改变了 /etc/inetd.conf 或者 /etc/services 文件后，运行 refresh -s inetd 或者 kill-1InetdPID 命令以告知 inetd 守护进程的改变到它的配置文件。

fingerd 守护进程应当有一个用户标识附带着至少的可能特权。nobody 标识允许最少的许可。赋予 fingerd 守护进程 nobody的用户标识后，它将允许守护进程能在您的主机上使用。改变 /etc/services 文件从而反映您想使用的用户标识。

利用系统资源控制器来操作 fingerd 守护进程

fingerd 守护进程是 inetd 守护进程的一个子服务器，它是 SRC 的一个子系统。fingerd 守护进程是 tcpip SRC 子系统组的一个成员。当这个守护进程没有在文件 /etc/inetd.conf 中注释时，它被启用，并且可以通过下面的 SRC 命令来操作：

startsrc 启动一个子系统、子系统组，或是一个子服务器。
stopsrc 终止子系统、子系统组或子服务器。
lssrc 获取一个子系统、子系统组或者一个子服务器的状态。

标志

-s 打开套接字级别调试。
-f 打开这个 fingerd 守护进程的 finger 转发服务。

示例

注:
通过使用 SMIT 或者通过编辑 /etc/inetd.conf 文件，fingerd 守护进程的参数可以被指定。
  1. 要启动 fingerd 守护进程，请输入： 

    startsrc -t finger

    这个命令启动 fingerd 子服务器。

  2. 要停止fingerd 通常的守护进程，请输入： 

    stopsrc -t finger

    这个命令允许所有的暂挂连接开始和存在连接完成，但是禁止新的连接开始。

  3. 要强行中止 fingerd 守护进程和所有的 fingerd 连接，请输入： 

    stopsrc -t -f finger

    这个命令立刻中止所有的暂挂连接和存在连接。

  4. 要显示一个关于 fingerd 守护进程的简短状态报告，请输入： 

    lssrc -t finger

    这个命令返回守护进程的名字、进程标识和状态（激活的和没有激活的）。
