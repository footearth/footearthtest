hosts2ldif 命令

用途

从一个主机文件中创建一个 LDAP 数据交换格式（LDIF）文件。

语法

hosts2ldif [ -i 输入文件 ] [ -o 输出文件 ] [ -s 搜索基址 ]

描述

/usr/sbin/hosts2ldif 命令从 /etc/hosts 或别的 类似于 /etc/hosts 的文件创建一个 LDAP 数据交换格式（LDIF）文件。没有标志，/etc/hosts 文件被用来创建 /tmp/hosts.ldif LDIF 文件，用 cn=hosts 作为基础域名。

使用这个命令创建的 LDIF 文件遵照 SecureWay(R) Directory Schema，可用来建立 ldap 机制。可支持 ldap 机制，推荐使用的是 nis_ldap 机制而不是 ldap 机制。

标志

-i 输入文件 指定用于输入的主机文件。
-o 输出文件 指定用于输出的 LDIF 文件。
-s 搜索基址 指定 LDAP 服务器上的主机表的基础域名。

示例

  1. 从 /etc/hosts 文件创建 /home/ldifhosts，请输入： 

    hosts2ldif -o /home/ldifhosts
  2. 从 /home/hosts.bak 文件创建 /tmp/hosts.ldif，请输入： 

    hosts2ldif -i /home/hosts.bak
  3. 使用 cn=hosttab 作为基础域名从 /etc/hosts 文件创建 /home/ldifhosts，请输入： 

    hosts2ldif -o /home/ldifhosts -s cn=hosttab

文件

/etc/hosts 包含在本地网络上的主机的网际协议（IP）名称和地址。
