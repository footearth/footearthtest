haemtrcoff 命令

用途

关闭事件管理器守护进程的跟踪.

语法

haemtrcoff -s subsys_name -a trace_list

描述

用 haemtrcoff 命令为事件管理器守护进程的指定活动关闭跟踪。将跟踪输出放在系统分区的事件管理跟踪日志中。

标志

-s subsys_name
指定事件管理子系统名。在一个节点上，这是 emsvcs。必须指定该参数。
-a trace_list
指定跟踪参数列表。每个参数指定活动类型，跟踪将针对该类型而关闭。至少必须指定一个参数。如果指定了一个以上的参数，则参数之间用逗号隔开。列表不包括空格。

参数

下列跟踪参数可能被指定：

init
停止跟踪事件管理器守护进程的初始化。
config
停止从配置文件转储信息。
insts
停止跟踪由守护进程处理的资源变量实例。
rmctrl
停止跟踪资源监视器控制。
cci
停止跟踪客户通讯（内部）接口。
emp
停止跟踪事件管理器协议。
obsv
停止跟踪资源变量观察。
evgn
停止跟踪事件的发生与通知。
reg
停止跟踪事件的注册和注销。
pci
停止跟踪对等的通讯（内部）接口。
msgs
停止跟踪守护进程收到以及发出的所有消息。
query
停止跟踪由守护进程处理的查询。
gsi
停止跟踪组服务接口（内部）。
eval
停止跟踪表达式求值。
rdi
停止跟踪可靠的守护进程接口（内部）。
sched
停止跟踪内部调度程序。
shm
停止跟踪共享存储器管理活动。
all
停止跟踪所有活动。
all_but_msgs
停止跟踪除消息外所有的活动. 消息活动由 msgs 参数定义.

安全性

您必须要 root 特权和对 SDR 的写权限以运行此命令.

您应该在控制工作站上运行这个命令. 在运行该命令前,您必须把 SP_NAME 这个环境变量设为正确的系统分区名.

退出状态

0
指示命令已成功完成.
1
显示有错误产生. 如果有一个或多个错误消息,则显示错误原因.

限制

在正常运行期间，别使用该命令. 只有在 IBM 技术支持中心指导下，才能使用该命令. 它提供了一些调试信息，并可能降低运行在系统分区中的事件管理子系统等的性能。

标准输出

当命令执行成功时，写入下列信息消息:

读分区syspar_name的事件管理数据
 
CDB=new_EMCDB_file_name Version=EMCDB_version_string

标准错误

必要时此命令把错误信息写到标准错误中。

示例

  1. 在一个群集节点上为事件管理子系统关闭所有跟踪，登录节点，请键入： 

    haemtrcoff -s emsvcs -a all
  2. 在一个群集节点上为事件管理子系统关闭所有初始化和配置的跟踪，登录该节点，请键入： 

    haemtrcoff -s emsvcs -a init,config

位置

/usr/sbin/rsct/bin/haemtrcoff
haemtrcoff 命令的位置

文件

/var/ha/log/em.trace.cluster_name
包括名为cluster_name 的群集上haemd 守护进程的跟踪日志。
/var/ha/log/em.msgtrace.cluster_name
包括来自名为 cluster_name 的群集上事件管理器守护进程的消息跟踪输出
