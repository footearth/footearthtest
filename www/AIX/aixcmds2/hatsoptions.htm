hatsoptions 命令

用途

在一个节点或一个控制工作站上控制拓扑服务选项。

语法

hatsoptions [-s] [-d]

描述

在此命令被执行之前，环境变量 HB_SERVER_SOCKET 必须被设置到被拓扑结构服务子系统所用的 UNIX 域套接字的位置。可使用下面的语句:

export HB_SERVER_SOCKET=/var/ha/soc/hats/server_socket.partition name

作为选择，变量 HA_SYSPAR_NAME 可被设置为分区名。

为使此命令成功执行，必须运行拓扑服务守护进程。

hatsoptions 可被用来控制拓扑服务中的许多选项。选项 -s 指示拓扑服务守护进程拒绝那些明显延迟的消息。这可在一个有时在网络或发送器与接收器节点间消息发生延迟的大系统配置中使用。仅当所有节点和控制工作站上的计时时钟都同步时才使用此选项。否则当发送器的计时时钟落后于接收器的计时时钟时，报文可能被错误地废弃。

选项 -d 指示拓扑服务守护进程不得拒绝那些明显延迟的报文。这个是缺省值。

标志

-s
指示拓扑服务守护进程拒绝那些明显延迟的报文。
-d
指示拓扑服务守护进程不得拒绝那些明显延迟的报文。（此为缺省设置）。

安全性

必须有 root 用户特权来运行此命令。

退出状态

0
指示命令已成功完成.
1
指示发生错误。

环境变量

HB_SERVER_SOCKET
此环境变量应在命令执行前进行设置。它必须被设置为被用来连接拓扑服务守护进程的拓扑服务客户机所用的 UNIX 域套接字的位置。此环境变量必须设置为 /var/ha/soc/hats/server_socket.partition name。
HA_SYSPAR_NAME
如果 HB_SERVER_SOCKET 没有进行设置，那么 HA_SYSPAR_NAME 必须被设置为分区名。

限制

此命令仅在对等域中有效。

标准输出

当-h flag 被指定时, 该命令的用法说明写到标准输出中。所有详细消息都写入标准输出中。

标准错误

必要时此命令把错误信息写到标准错误中。

示例

为指示本地节点上的拓扑服务守护进程开始废弃明显延迟的报文，输入:

export HA_SYSPAR_NAME=partition1
 
/usr/sbin/rsct/bin/hatsoptions -s

位置

/usr/sbin/rsct/bin/hatsoptions
包含 hatsoptions 命令

文件

/var/ha/soc/hats/server_socket.partition name

相关信息

命令：hatsctrl、hats、lssrc、startsrc、stopsrc、 syspar_ctrl。
