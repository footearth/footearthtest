dtappintegrate 命令

用途

公共桌面环境应用程序集成工具。

语法

dtappintegrate -s ApplicationRoot [ -t TargetPath ] [-l Language ] [ -u ]

描述

dtappintegrate 命令从专用位置链接应用程序 CDE 配置文件到系统位置，并且为受影响的语言更新系统浏览器帮助卷。在应用程序的安装过程期间使用 dtappintegrate 命令。应用程序安装脚本应该在末尾调用 dtappintegrate 命令。

在应用程序根目录下有四个由 CDE 策略规定的关键子目录（请参考 $APP_ROOT）。这些目录有：

$APP_ROOT/dt/appconfig/types/Language 对于文件类型、前面板和操作文件。
$APP_ROOT/dt/appconfig/appmanager/Language 对于应用程序组文件。
$APP_ROOT/dt/appconfig/icons/Language 对于 CDE 管理器使用的图标
$APP_ROOT/dt/appconfig/help/Language 对于应用程序帮助。例如，缺省语言应用程序 SpreadSheet 将在 /opt/SpreadSheet/dt/appconfig/icons/C/*.bm 和 /opt/SpreadSheet/dt/appconfig/icons/C/*.pm 下装入桌面图标，其中 /opt/SpreadSheet 是 $APP_ROOT 的值。

注：$APP_ROOT 是这个文档的一个语法约定，并且运行时环境没有使用它。所有这些 CDE 配置文件和子目录都放在公共上边框的下面，并且应该始终包含缺省语言子目录 C。

在最简单的情况下，命令作为输入应用程序根目录，例如，/opt/thisapp。从这一操作的输出是应用程序服务器上的相应子目录和文件，在下面的系统位置上，它们包含如上描述的应用程序 CDE 配置文件的相关符号链接：

/etc/dt/appconfig 顶级应用程序配置子目录是由下列子目录组成的：
/etc/dt/appconfig/types/语言/
包含*.dt以及任何*.fp链接。
/etc/dt/appconfig/appmanager/语言/
在应用程序管理器下，包含到应用程序组子目录的链接，以及到以操作出现的操作脚本文件的链接。
/etc/dt/appconfig/help/语言/
包含到安装在应用程序根目录下帮助文件的符号链接。
/etc/dt/appconfig/icons/语言/
包含到应用程序的 CDE 图标的符号链接。

标志

-s ApplicationRoot 集成位于 ApplicationRoot 上的应用程序。这一标志是必需的。
-t TargetPath 从特定应用程序位置的应用程序 CDE 配置文件链接到 TargetPath，而不是链接到系统位置。这一标志是可选的。

如果提供了 -t 标志，文件在指定子目录下链接。例如，指定 -t /etc/dt/private 将会导致在 /etc/dt/private/help/Language 下符号地链接应用程序帮助文件。通常只有想创建独立应用程序的管理员才会使用这一标志，在应用程序安装后脚本中并不使用。通过缺省值（没有指定 -t），应用程序子目录的根目录对应用程序主机是全局的。安装在主机上的所有应用程序将它们的配置文件复制到合并其它应用程序配置文件的相同地方。
-l Language 指定要集成的语言。基本上，这一标志指明了在哪些目录里可以查找应用程序 CDE 的配置文件。如果没有指定这一参数，所有的语言都将集成。这一参数是可选的。
-u 取消应用程序集成。这一标志是可选的。
