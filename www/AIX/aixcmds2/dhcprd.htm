dhcprd 守护进程

用途

转发本地网络的 BOOTP 和 DHCP 分组。

语法

用系统资源控制器向 DHCP 服务器转发信息：

startsrc -s dhcprd [ -a Argument ] [ -a Argument ] ...

不用系统资源控制器向 DHCP 服务器转发信息：

dhcprd [ -f ConfigurationFile ]

描述

dhcprd 守护进程侦听、接收广播分组，并转发它们到适当的服务器。这将可以避免广播分组必须传到其他网络的要求。DHCP 中继代理负责将 DHCP 和 BOOTP 客户机广播分组从本地网络传到一组服务器。BOOTP 或者 DHCP 客户机发送的初始分组是客户机所在的本地网络接口上的广播。这些分组不允许通过网关和路由器。作为 BOOTP/DHCP 中继代理，dhcprd 守护进程将把这些分组传给适当的服务器。

DHCP 服务器读取 /etc/services 文件以决定使用哪个端口来接收请求。缺省服务为 dhcps。因为这是与 bootpd 守护进程所用的同样的端口，您只能运行一个（dhcprd 或者 bootpd）守护进程。如果您选择 dhcprd 守护进程，您就需要从 /etc/inetd.conf 文件中取消注释 bootp，接着在命令行输入 refresh -s inetd。

注:
如果运行了 bootpd，则该程序在启动守护进程前必须停止。

标志

-f ConfigurationFile 指定要用的配置文件。缺省值为 /etc/dhcpcd.ini 文件。

退出状态

此命令返回下列退出值：

0 成功结束。
>0 发生错误。

安全性

访问控制：您必须拥有根权限来运行这个命令。

文件

/usr/sbin/dhcprd 包含 dhcprd 守护进程。
/etc/dhcprd.cnf 包含缺省配置文件。
/etc/services 定义用于网络服务的套接字和协议。
/etc/inetd.conf 定义 inetd 守护进程控制的服务。
