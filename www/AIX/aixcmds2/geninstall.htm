geninstall 命令

用途

通用安装程序，用于安装多种打包格式的软件产品。如 installp、RPM 和 ISMP。

语法

geninstall -d Media [ -I installpFlags ] [ -E | -T | -R ] [ -t ResponseFileLocation ] [-e LogFile] [ -p ] [ -F ] [ -Y ] [ -Z ] { -f File | Install_List ] | all}

或

geninstall -u [-e LogFile] [ -E | -T | -R ] [ -t ResponseFileLocation ] {-f File | Uninstall_List...}

或

geninstall -L -d Media [-e LogFile]

描述

接受当前所有的 installp 标志，并把它们传到 installp 上去。一些标志(如，-L)会过载来表示列出介质上所有的产品。对 ISMP 打包产品没有意义的标志被忽略。这允许程序（如 NIM）继续一直向 geninstall 发送 installp 标志, 但仅使用有意义的标志。

一个 geninstall 命令的安装活动摘要保存在/var/adm/sw/geninstall.summary中。该文件含有由 installp 安装的文件集列表（这些表用冒号分隔），以及用 ISMP 安装的组件。这主要用来提供无提示安装的摘要信息。

注:
参考 README.ISMP 文件，该文件在 /usr/lpp/bos 目录下，从该文件中可以了解到更多关于 ISMP 打包的安装和使用响应文件的信息。

标志

-d 设备或目录 指定包含有要安装映像的设备或目录。
-e LogFile 启用事件记录日志。-e 标志使用户能够把geninstall 命令输出的某些部分附加到由 LogFile 变量指定的文件中去。LogFile 必须指定一个已存在的可写文件，并且该文件驻留的文件系统必须有足够空间来存储日志。日志文件不折行。
-E 在缺省位置（包含产品安装文件的目录）创建的 ISMP 响应文件记录。该选项要求交互式、完全地运行 ISMP 安装或卸载程序。生成的响应文件可和 -R 标志一起使用以在将来对同一产品进行安装或卸载时选择相同的选项。创建响应文件记录也导致产品的安装或卸载。
-f File 指定包含要安装条目列表的文件。ISMP 打包产品必须含有一个 J:，后面跟产品安装程序或子目录。RPM 和 ISMP 映像应分别带有“R:”或“J:”的前缀。efix 和 emgr 映像应带有前缀“E:”。installp 映像应带有前缀“I:”。这个信息在 geninstall -L 输出中给出 (如, J:product_directory or installer 或 J:WebSphere)。用于 emgr/efix 映像的目录路径为：emgr/ppc。
-F 允许用户对已经安装的包重新安装，也可安装比当前已安装版本更旧的包。
-I installpFlags 指定installp 所使用的标志，当调用 installp 命令时。installp 进行安装期间所使用的标志是a, b, c, D, e, E, F, g, I, J, M, N, O, p, Q, q, S, t, v, V, w 和 X 等标志。The installp flags that are not used during install are the C, i, r , z, A, and l flags. installp 命令应该直接调用以执行这些功能。-u, -d, -L 和 -f 标志应无 -I 标志时使用。
-L 对介质上的内容进行列表。输出格式与 installp -Lc 格式相同, 对于用 ISMP 和 RPM 格式化的产品，末尾有附加字段。
-N 用 no-op 来调用 ISMP 安装程序或预览 ISMP 安装程序的模式。
-p 通过运行指定操作的所有预安装检测程序来执行一个预览操作。
-R 通过将缺省位置的响应文件传递到指定 ISMP 封装的产品的 ISMP 安装程序来向可配置选项提供预先确定的响应。缺省位置是包含产品安装文件的目录。
-t ResponseFileLocation 允许指定响应文件或响应文件模板的备用位置。缺省位置是包含产品安装文件的目录。如果期望在其它位置创建响应文件记录或模板，或如果期望将响应文件从非缺省位置传递到 -R 标志，则可以使用此标志。ResponseFileLocation 可以是文件或目录名。如果 ResponseFileLocation 是目录，它必须已经存在。如果 ResponseFileLocation 不是现有目录，则将假定为指定了文件名。
-T 在缺省位置（包含产品安装文件的目录）创建的 ISMP 响应文件模板。生成的模板用在将来采用您所选的选项来安装或卸载同一产品时创建响应文件。响应文件模板的创建将不会引起产品的安装或卸载。
-u 执行对指定软件的卸载。对 ISMP 产品，调用在供应商数据库中列出、前缀为 "J:"的卸载程序。
-Y 同意软件安装需要的软件许可证协议。该标志可接受为 -I 选项的 installp 标志。
-Z 通知 geninstall 在无提示模式下调用安装。

例子

安装在驱动器 cd0 中的 CD 介质的所有产品，请输入：

geninstall -d /dev/cd0 all

如果 ISMP 映像在介质上存在, 则显示一个图形化界面。无提示安装任何一个installp 或 RPM 映像，除非installp 映像展开在多个 CD 上。

文件
