haemd 守护程序

用途

查看资源监视器更新的资源变量实例，并生成事件和向客户机程序报告事件。

语法

haemd

描述

haemd（事件管理器）守护进程可查看资源监视器更新的资源变量实例，并生成事件和向客户机程序报告事件。

在群集的每一个节点上都执行着一个 haemd 守护进程的实例。haemd 守护进程受系统资源控制器（SRC）的控制。

由于受 SRC 的控制，该守护进程不能直接从命令行启动。它通常由 emsvcsctrl 命令启动。如果您必须要直接启动或停止该守护进程，请使用 emsvcsctrl 命令。

在 SRC 创建 haemd 守护进程时，实际启动的程序是 haemd_HACMP。haemd_HACMP 程序首先会收集 haemd 守护进程所需的信息，然后才执行 haemd 程序。换言之，haemd_HACMP 程序将在由 SRC 创建的进程中被替代成 haemd 程序。

有关该事件管理器守护进程的更多信息，请参阅 emsvcsctrl 命令。

标志

-a
添加子系统。
-s
启动子系统。
-k
停止子系统。
-d
删除子系统。
-c
清除子系统，即从所有系统分区中将它们删除。
-u
从所有系统分区取消对子系统的配置。
-t
启用子系统跟踪功能。
-o
禁用子系统的跟踪功能。
-r
刷新子系统.
-h
显示用法信息.

参数

无。

安全性

要运行这个命令，您必须有 root 用户特权和对 SDR 的写访问权。

退出状态

0
指示命令已成功完成.
1
显示有错误产生.

限制

此命令只在 PSSP 环境下有效.

标准输出

当命令执行成功时，写入下列信息消息:

读分区 syspar_name 的事件管理数据
 
CDB=new_EMCDB_file_name Version=EMCDB_version_string

标准错误

必要时此命令把错误信息写到标准错误中。

示例

  1. 向当前系统分区的 SRC 添加事件管理子系统，请将 SP_NAME 环境变量设置成相应系统分区的名称，然后输入： 
    haemctrl -a
  2. 启动当前系统分区中的事件管理子系统，请将 SP_NAME 环境变量设置成相应系统分区的名称，然后输入： 
    haemctrl -s
  3. 停止当前系统分区中的事件管理子系统，请将 SP_NAME 环境变量设置成相应系统分区的名称，然后输入： 
    haemctrl -k
  4. 从当前系统分区的 SRC 中删除事件管理子系统，请将 SP_NAME 环境变量设置成相应系统分区的名称，然后输入： 
    haemctrl -d
  5. 清理所有系统分区中的事件管理子系统，请输入： 
    haemctrl -c
  6. 在控制站上从所有系统分区中取消对事件管理子系统的配置，请输入： 
    haemctrl -u
  7. 启用当前系统分区中事件管理守护进程的跟踪功能，请将 SP_NAME 环境变量设置成相应系统分区的名称，然后输入： 
    haemctrl -t
  8. 关闭当前系统分区中事件管理守护进程的跟踪功能，请将 SP_NAME 环境变量设置成相应系统分区的名称，然后输入： 
    haemctrl -o

位置

/usr/sbin/rsct/bin/haemd
haemd 守护进程所在的位置

文件

/var/ha/log/em.default.cluster_name
包含名为 cluster_name 的群集上的 haemd 守护进程的缺省日志。
/var/ha/log/em.cp
包含在复制事件管理配置数据库时所产生的任何错误的日志。
/var/ha/log/em.trace.cluster_name
包含名为 cluster_name 的群集上的 haemd 守护进程的跟踪日志。
/var/ha/log/em.mkgroup
包含在创建 haemrm 组时所产生的任何错误的日志。
/var/ha/log/em.mkdir
包含在创建 /var/ha/lck/haem 和 /var/ha/soc/haem 目录时所产生的任何错误的日志。
