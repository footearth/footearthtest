dhcpcd 守护进程

用途

实现动态主机配置协议（DHCP）客户机。服务器地址和 DHCP 服务器的配置信息。

语法

通过系统资源控制器实现一个 DHCP 客户机：

startsrc -s dhcpcd [ -aArgument] ...

不通过系统资源控制器实现一个 DHCP 客户机：

dhcpcd [ -f ConfigurationFile] [ -i IPAddress] [ -t Seconds ] [ -T Minutes ]

描述

dhcpcd 守护进程通过设置 IP（网际协议）地址来实现 DHCP 客户机，并通过使用 DHCP 协议来实现其它参数。

dhcpcd 守护进程正常情况下由通常运行于系统引导时的 /etc/rc.net 文件启动。在缺省情况下，这将被注释掉，而且不在机器启动时运行。smit 选项可以启用 DHCP 客户机。

dhcpcd 守护进程读取它的配置文件，并在配置文件中试图提出并得到一个 IP 地址，以及其他针对特定接口的配置选项。dhcpcd 守护进程在系统启动后运行于后台。它将根据需要更新一个已经接收的地址。

在使用 -i 标志时，dhcpcd 守护进程也以 DHCP Inform 方式运行。该方式允许客户机在没有 IP 地址的情况下检索服务器 DHCP 上的配置信息。这对静态地址很有用，但是对诸如打印服务器等动态项和其他选项没用。dhcpcd 守护进程 对于指定地址将运行一次。

refresh 命令可以使 dhcpcd 守护进程重新读取配置文件。SIGHUP 也可用于获取相同的响应。

dhcpcd 的缺省配置文件是 /etc/dhcpcd.ini。它包含日志和网络接口信息。

您可以通过 基于 Web 的系统管理器 应用程序（wsm network 快速路径）来运行这个命令。您也可以使用系统管理接口工具（SMIT）smit usedhcp 快速路径来运行该命令。

标志

-f ConfigurationFile 指定要用的配置文件。缺省为 /etc/dhcpcd.ini 文件。
-i IPAddress 指定 dhcpcd 守护进程使用 DHCP Inform 方式。IP 地址告诉 DHCP 到哪个接口去获取配置信息。
-t Seconds 指定 dhcpcd 置其自身于后台之前将等待的秒数。如果没有找到 DHCP 服务器，这允许机器继续引导。
-T Minutes 指定时间（以分钟为单位）。如果 dhcp 客户机无法在此超时值内为接口配置地址（例如，由于 dhcp 服务器不可用），它将停止进一步的尝试。

退出状态

此命令返回下列退出值：

0 成功结束。
>0 发生错误。

安全性

访问控制：您必须拥有 root 用户权限来运行这个命令。

文件

/usr/sbin/dhcpcd 包含 dhcpcd 守护进程。
/etc/dhcpcd.ini 包含缺省客户机配置文件
/etc/services 定义用于网络服务的套接字和协议。
/etc/inetd.conf 定义 inetd 守护进程控制的服务。
