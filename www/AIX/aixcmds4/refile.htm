refile 命令

用途

在文件夹间移动文件。

语法

refile [ -src +Folder ] [ -draft ] [ -file File ] [ Messages ] [ -nolink | -link ] [ -nopreserve | -preserve ] +Folder ...

描述

refile 命令在文件夹中移动消息。如果不指定源文件夹，则 refile 命令使用当前文件夹作为源文件夹。如果指定的目标文件夹不存在，系统请求创建它的许可权。

refile 命令也把消息从一个文件夹复制到另一个文件夹。移动消息时，缺省情况下，系统并不在原始文件夹中保留消息的副本。要保留副本，使用 -preserve 标志。

标志

-draft 从邮件目录中复制当前草稿消息。
-file File 复制指定的文件。文件必须具有有效的消息格式。 使用 inc 命令正确地格式化新消息并把它归档。
+Folder 复制消息到指定的文件夹。可以指定任意多个文件夹。
-help 列出命令语法、可用的参数选项符（切换）和版本信息。

注： 对于 MH，该标志的名称必须全拼写。
-link 消息被复制后，把它们放到源文件夹或文件。
消息 指定要复制的消息。可以指定几个消息、一个范围内的消息、或一个单独的消息。使用指定消息的下列引用：
Number
消息号
Sequence
用户指定的一组消息。可标识的值包含：
all
文件夹中全部消息。
cur or . (period)
当前消息。这是缺省值。
first
文件夹中的第一条消息。
last
文件夹中的最后一条消息。
next
在当前消息后的消息。
prev
在当前消息前的消息。
 
 /DT>
如果 -link 和 all 标志一起使用，当前文件夹中的当前消息不发生改变。 否则，如果指定消息，则重新归档的消息变成当前消息。
-nolink 消息被复制后，从源文件夹或文件中除去它们。此标志是缺省的。
-nopreserve 给复制的消息重新编号。重新编号开始号码比目标文件夹的最后一条消息的号码高 1。此标志是缺省的。
-preserve 保留所复制消息的消息号。如果这些号码的消息已经存在，则 refile 命令发出一个错误消息，并不改变文件夹的内容。
-src +Folder 标识源文件夹。缺省情况下，系统使用当前文件夹。

概要文件条目

以下条目是 UserMHDirectory/.mh_profile 文件的一部分：

Current-Folder: 设置缺省的当前文件夹。
Folder-Protect: 为新的文件夹目录设置保护级别。
Path: 指定 UserMhDirectory。
rmmproc: 指定用于从文件夹中除去消息的程序。

示例

  1. 重新归档来自当前文件夹的消息并把它放到一个称为 meetings 的新文件夹中，请输入：
    refile  +meetings
    系统用类似下列的消息响应： 

    Create folder "/home/jeanne/Mail/meetings"?

    输入 y 以创建文件夹。当前文件夹中不保留原始消息的副本。

  2. 要把来自当前文件夹的当前消息复制到 meetings 文件夹中，请输入：
    refile  -link +meetings
    原始消息保留在当前文件夹中。
  3. 把当前消息草稿重新归档到 test 文件夹，请输入：
    refile  -draft +test
    当前文件夹中不保留消息草稿的副本。
  4. 重新归档来自当前文件夹的消息到几个文件夹，请输入：
    refile  +tom +pat +jay
    当前文件夹中不保留该消息的副本。

文件

$HOME/.mh_profile 设置 MH 用户概要文件。
/usr/bin/refile 包含 refile 命令。
