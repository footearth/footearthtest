niminit 命令

用途

配置网络安装管理（NIM）客户机软件包。

语法

配置 NIM 客户机软件包

niminit{-a name=Name -a pif_name=Pif -a master=Hostname} [ -a master_port=PortNumber ] [ -a registration_port=PortNumber ] [ -a cable_type=Type | -a ring_speed=Speed] [-a iplrom_emu=Device ] [ -a platform=PlatformType ] [ -a netboot_kernel=NetbootKernelType ] [-a adpt_add=AdapterAddress]

重建 /etc/niminfo 文件

niminit {-a name=Name -a master=Hostname -a master_port=PortNumber}

描述

niminit 命令配置 NIM 客户机软件包。这必须在 nimclient 命令能够使用前完成。当必需的属性被提供给 niminit 命令时，将创建一个新的机器对象来表示执行 niminit 命令的机器。当 niminit 命令成功完成时，该机器将能够参与到 NIM 环境中。

成功配置 NIM 客户机软件包后，niminit 命令就能再次运行以在客户机上重建 /etc/niminfo。nimclient 命令使用 /etc/niminfo 文件，如果该文件被某用户意外的除去，就必须重建该文件。

标志

-a 为 niminit 命令指定多达五个不同的属性。所有下列成对的 attribute=value 前都附有 -a 标志：
name=Name 指定 NIM 用于标识工作站的名称。该值是必需的。
pif_name=Pif 为全部的 NIM 通信定义网络接口的名称。该值是必需的。
master=Hostname 指定 NIM 主控机的主机名。客户机必须有能力将该主机名解析为网际协议（IP）地址。该值是必需的。
master_port=PortNumber 指定用于 NIM 通信的 nimesis 守护程序的端口号。
cable_type=CableType 指定以太网电缆类型。当 pif_name 指的是一个以太网，那么该值必须给定。可接受的值有：bnc、dix 和 N/A。
ring_speed=Speed 速度以兆比特每秒计。当 pif_name 指的是一个令牌环网络，那么该值必须给定。可接受的值有：4 和 16。
iplrom_emu=Device 指定一个包含 ROM 模拟映像的设备。该映像对于内部不支持通过网络接口来引导的模型是必需的。
platform=PlatformType 指定符合客户机机器类型的平台。如果未指定该属性，则使用缺省值 rs6k。支持的平台有:
rs6k
基于微通道、单处理机模型（对于 AIX 5.1 以及更早的版本）
rs6ksmp
基于微通道、对称多处理机模型（对于 AIX 5.1 以及更早的版本）
rspc
基于 PowerPc 微处理器 PCI 总线、单处理机模型（对于 AIX 5.1 以及更早的版本）
rspcsmp
基于 PowerPc 微处理器 PCI 总线、对称多处理机模型（对于 AIX 5.1 以及更早的版本）
adpt_add=AdapterAddress 指定与网络适配器相符合的硬件地址。
registration_port=PortNumber 指定用于 NIM 客户机注册的端口号。
注:
  1. 如果您未在命令行指定端口号，那么就使用 /etc/services 文件中 NIM 的端口号。 如果 /etc/services 文件未包含 NIM 端口 nim 和 nimreg 的项，那么使用 master_port 的缺省值 1058 以及 registration_port 的缺省值 1059。
  2. master_port 和 registration_port 所使用的值应与 NIM 主控机使用的值相匹配。要显示 NIM 主控机使用的值，在 NIM 主控机上运行命令 lsnim -l master。
netboot_kernel= NetbootKernelType 当从网络上引导客户机时，请指定要使用的内核类型。netboot_kernel 的值是：
up
单处理机的内核。
mp
多处理机的内核。

缺省值是 up。

安全性

访问控制：您必须拥有运行 niminit 命令的 root 权限。

示例

  1. 要在具有 IPL ROM（启用 BOOTP）的机器上配置 NIM 客户机软件包，使该软件包在 NIM 环境中被认为是 scuba，使用 en0 作为其主接口以及 bnc 的太网网电缆类型，指定使用 manta 的主控机的主机名以及位于 /etc/services 的缺省 NIM 端口（用于网络安装通信）与 NIM 主控机通信，请输入： 

    niminit -a name=scuba -a pif_name=en0 -a cable_type=bnc \
-a master=manta
  2. 当 /etc/niminfo 文件被用户意外除去时，要使用用于主控机主机名的 superman 主机名以及端口号 1058 重建该文件，请输入： 

    niminit -a name=robin -a master=superman -a master_port=1058
  3. 对于 AIX 5.1 及更早的版本，在基于 PowerPC 微处理器 PCI 总线的单处理机系统并且具有 IPL ROM（启用 BOOTP）的机器上配置 NIM 客户机软件包，使该软件包在 NIM 环境中被认为是 starfish ，使用 en0 作为其主接口以及 dix 的以太网电缆类型，指定使用主控机的主机名 whale 以及端口号 1058 与 NIM 主控机通信，请输入： 

    niminit -a name=starfish -a pif_name=en0 -a cable_type=dix \
-a master=whale -a master_port=1058 -a platform=rspc
  4. 要在 NIM 环境中被认为是 bluefish 的机器上配置 NIM 客户机，使用 at0 作为其主接口，指定使用主控机的主机名 redfish 以及端口号 1058 与 NIM 主控机通信，请输入： 

    niminit -a name=bluefish -a pif_name=at0 -a master=redfish \
-a master_port=1058
    注:
    由于 ATM 网络的接口当前不支持通过网络引导，如果已定义了一个与客户机子网相符的通用网络对象，那么该操作将在 NIM 主控机上定义一个机器对象。
  5. 对于 AIX 5.1 及更早的版本，要在基于 PowerPC 微处理器 PCI 总线的对称多处理机系统并且具有 IPL ROM（启用 BOOTP）的机器上配置 NIM 客户机，使其在 NIM 环境中被认为是 starfish ，使用 en0 作为其主接口以及 dix 的以太网电缆类型，指定使用主控机的主机名 whale 以及端口号 1058 与 NIM 主控机通信，请输入： 

    niminit -a name=jellyfish -a pif_name=en0 -a cable_type=dix \
-a master=whale -a master_port=1058 -a platform=rspcsmp
  6. 要在设备 /dev/fd0 中使用 IPL ROM 仿真的机器上配置 NIM 客户机软件包，使其被认为是在 NIM 环境中的 octopus，使用 tr0 作为其主接口以及环速度 16，使用主控机的主机名 dolphin 以及端口号 1700（用于客户机通信）和端口号 1701（用于客户机注册）与 NIM 主控机通信，请输入： 

    niminit -a iplrom_emu=/dev/fd0 -a name=octopus -a pif_name=tr0 \
-a ring_speed=16 -a master=dolphin -a master_port=1700 \
-a registration_port=1701
