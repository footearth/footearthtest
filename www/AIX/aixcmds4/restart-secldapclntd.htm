restart-secldapclntd 命令

用途

restart-secldapclntd 脚本用来停止当前 secldapclntd 守护程序进程的运行然后重新启动。

语法

/usr/sbin/restart-secldapclntd [ -C CacheSize ] [ -p NumOfThread ] [ -t CacheTimeOut ] [ -T HeartBeatIntv ]

描述

restart-secldapclntd 脚本将停止 secldapclntd 守护程序（如果其正在运行），然后重新启动。如果 secldapclntd 守护程序没有运行，则仅启动它。

标志

缺省情况下，secldapclntd 守护程序读取启动时 在 /etc/security/ldap/ldap.cfg 文件中指定的配置信息。当启动 secldapclntd 进程时，如果下面的选项在命令行给出，命令行的选项将覆盖 /etc/security/ldap/ldap.cfg 文件中的值。

-C CacheSize 设置 secldapclntd 守护程序使用的最大高速缓存项数为 CacheSize 项数。用户高速缓存的有效范围是 100-10,000 项。缺省值是 1000。 组高速缓存项将是用户高速缓存项的 10％。
-p NumOfThread 设置 secldapclntd 守护程序使用的线程数至 NumOfThread 线程。有效的范围是 1-1000。 缺省值是 10。
-t CacheTimeout 设置高速缓存在 CacheTimeout 秒后失效。有效的范围是 60 - 3600 。 缺省值是 300 秒。
-T HeartBeatIntv 在此客户机和 LDAP 服务器之间设置波动信号的时间间隔。有效的值是 60 - 3,600 秒。缺省值是 300 秒。

示例

  1. 重新启动 secldapclntd 守护程序，请输入： 
    /usr/sbin/restart-secldapclntd
  2. 用 30 个线程和高速缓存超时值 500 秒重新启动 secldapclntd，请输入： 
    /usr/sbin/restart-secldapclntd -p 30 -t 500

文件

/etc/security/ldap/ldap.cfg 包含 secldapclntd 守护程序连接到服务器所需要的信息。
