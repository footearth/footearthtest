rmcctrl 命令

用途

管理资源监视和控制（RMC）子系统。

语法

rmcctrl {-a | -A | -d | -k | -K | -m {-R | -E | -D} | -p | -P | -q | -Q | -s | -z | -h}

描述

rmcctrl 命令控制资源监视和控制（RMC）子系统的操作。带有一个名为 ctrmc 的子系统及一个名为 rsct 的子系统组的系统资源控制器（SRC）控制此子系统。RMC 子系统定义被添加到子系统对象类然后当安装了可靠的可伸缩群集技术（RSCT）时启动。另外，在 /etc/inittab 文件中创建一项，使 RMC 子系统在系统引导时自动启动。

注：虽然 RMC 子系统可以通过使用 stopsrc 和 startsrc 命令来停止和启动，建议使用 rmcctrl 命令来执行这些功能。

标志

-a
将 RMC 子系统添加到子系统对象类，并在 /etc/inittab 文件末尾放置项。
-A
添加并启动 RMC 子系统。
-d
从子系统对象类中删除 RMC 子系统，并从 /etc/inittab 文件中除去 RMC 项。
-k
停止 RMC 子系统。
-K
停止 RMC 子系统和所有资源管理器。
-m
指定 RMC 子系统客户机消息策略。该策略应用于在 RMC 子系统和在 IBM RSCT for AIX 5L: Technical Reference 中列出的很和命令（当该命令在与 RMC 子系统不同的节点上运行时，也就是说，设置了 CT_CONTACT 环境变量）之间发送的消息。使用 TCP/IP 发送这些消息。

该标志仅在 RSCT V2.3.1.0 或更新版本中受支持。如果命令出自早期版本的 RSCT，则必须使用“Enabled”策略。

R
表明客户机消息策略为“Required”。“Required”意味着只有可以（将要）使用消息认证连接才保留打开。
E
表明客户机消息策略为“Enabled”。“Enabled”是缺省值；如果连接的两端都支持它，则使用消息认证。
D
表明客户机消息策略为“Disabled”。“Disabled”意味着未使用消息认证。
-p
启用远程客户机连接。
-P
禁用远程客户机连接。
-q
在 RMC 子系统下次启动时启用远程客户机连接。
-Q
在 RMC 子系统下次启动时禁用远程客户机连接。
-s
启动 RMC 子系统。
-z
停止 RMC 子系统和所有资源管理器，但是该命令在 RMC 子系统和资源管理器真正停止前不会返回。
-h
将命令的用法语句写到标准输出。

安全性

特权控制：只有 root 用户拥有到此命令的执行（x）访问权。

退出状态

0
命令已经成功运行。
1
命令不成功。

限制

本命令只适用于 cthags 和 cthats 子系统。

标准输出

当指定 -h 标志时，此命令的用法语句将被写入标准输出。

标准错误

将错误消息写到标准错误（并写到 ctsnap.host_name.nnnnnnnn.log 文件）。

示例

  1. 要添加 RMC 子系统，输入： 

    rmcctrl -a
  2. 要启动 RMC 子系统，输入： 

    rmcctrl -s
  3. 要停止 RMC 子系统，输入： 

    rmcctrl -k
  4. 要删除 RMC 子系统，输入： 

    rmcctrl -d

位置

/usr/sbin/rsct/bin/rmcctrl
包含 rmcctrl 命令

文件

/tmp/ctsupt
包含输出文件的缺省目录的位置。
/tmp/ctsupt/ctsnap.host_name.nnnnnnnn.log
命令执行的日志文件的位置，在那里 nnnnnnnn 是时间戳记并且 host_name 是运行此命令的主机名。
tmp/ctsupt/ctsnap.host_name.nnnnnnnn.tar.Z
包含收集数据的压缩归档文件的位置，在那里 nnnnnnnn 是时间戳记并且 host_name 是运行此命令的主机名。
