qhld 命令

用途

保留和释放假脱机打印作业。

语法

qhld [  -r ] {  -#JobNumber [  -PQueue ] | -PQueue | -uUser [  -PQueue ] }

描述

qhld 命令保留在假脱机状态下的打印作业。要保留的作业是由作业号、队列或用户名指定的。-r 标志释放保留的打印作业。

在 AIX 4.3.2 和更高版本中，已增强 qstatus 以改善对显示重复的三数字的作业号的本地队列的管理。在 enq、qchk、lpstat、lpq 状态命令中使用 -W 标志以显示更多的作业号数字。

如果队列显示出重复的三数字作业号，使用 qchk -W 可以更大的精度列出作业号。然后可保留特定的作业。

例如，qchk 两次显示作业号 123，qchk -W 显示作业号 1123 和 2123。如果想保留作业号 2123，指定 qhld -# 123，将导致 qdaemon 保留它在内部列表中找到的第一个匹配的作业号，它可能是 1123。通过 qstatus -W 提供的附加信息，可以保留特定的作业号。

标志

-#JobNumber 指定要保留的打印作业号。
-PQueue 指定要保留的打印队列。
-r 释放编号、队列、或用户名指定的打印作业。
-uUser 指定将保留其打印作业的用户的名称。

示例

  1. 要保留打印作业号 300，请输入： 

    qhld -#300
  2. 要保留队列 lp0 上的所有打印作业，请输入： 

    qhld -P lp0
  3. 要保留所有属于用户 fred 的作业，请输入： 

    qhld -u fred
  4. 要释放作业号 300，请输入： 

    qhld -#300 -r
  5. 要释放队列 lp0 上的所有作业，请输入： 

    qhld -Plp0 -r
  6. 要释放属于用户 fred 的所有作业，请输入： 

    qhld -u fred -r

文件

/usr/sbin/qdaemon 包含 qdaemon 守护程序。
/var/spool/lpd/qdir/* 包含作业描述文件。
/var/spool/lpd/stat/* 包含关于设备状态的信息。
/var/spool/qdaemon/* 包含入队文件的临时副本。
/etc/qconfig 包含配置文件。
/etc/qconfig.bin 包含 /etc/qconfig 文件的摘要二进制版本。
