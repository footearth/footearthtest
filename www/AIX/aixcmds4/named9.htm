named9 守护程序

用途

因特网域名服务器。

语法

named9 [ -c config-file ] [ -d debug-level ] [ -f ] [ -g ] [ -n #cpus ] [ -p port ] [ -s ] [ -v ] [ -x cache-file ]

描述

named9 是域名系统（DNS）服务器，是 ISC 中的 BIND 9 分配的部件。更详细的关于 DNS 信息，请参阅 RFC 1033、1034 和 1035。如调用时无参数，named 将读取缺省配置文件 /etc/named.conf，读取任意初始数据，并侦听查询。

标志

-cconfig-file 使用 config-file 作为配置文件替代缺省文件 /etc/named.conf。为确保重新装入配置文件在服务器由于配置文件中可能的目录选项而更改其工作目录后可继续工作，配置文件的目录应为绝对路径名。
-ddebug-level 将守护程序的调试级别设置为 debug-level。当调试级别增加时，named 的调试跟踪将变得更为详细。
-f 在前台运行服务器（例如，非 daemonize）。
-g 在前台运行服务器并强制记录全部至标准错误。
-n #cpus 创建 #cpus 工作程序多线程，以利用多个 CPU。如未指定，named 将尝试确定现有的 CPU 数目，并为每个 CPU 创建一个线程。如果 CPU 数目不可确定，则创建单一工作程序线程。
-pport 侦听端口 port 上的查询。如未指定，缺省值为端口 53。
-s 退出时，内存使用统计信息写入 stdout。
注:
本选项主要对 BIND 9 开发者有意义，且在以后的发行版中有可能被除去或更改。
-v 报告版本号并退出。
-x cache-file 从高速缓存文件中将数据装入缺省视图的高速缓存。
注意: 本选项并非必须使用。它只对 BIND 9 开发者有意义。

信号

在例程操作中，信号不应用以控制名称服务器；应用 rndc 替代。

SIGHUP 强制服务器重新装入。
SIGINT, SIGTERM 关闭服务器。

发送任何其它信号至服务器，其结果不确定。

配置

named9 配置文件的完整描述在《BIND 9 管理员参考手册》中有提供。

文件

/usr/sbin/named9 包含 named9 守护程序。
/etc/named.conf 缺省配置文件。
/etc/named.pid 缺省进程标识文件。
