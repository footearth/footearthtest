reset 命令

用途

初始化终端。

语法

reset [ -e C ] [ -k C ] [ -i C ] [ - ] [ -s ] [ -n ] [ -I ] [ -Q ] [ -m [ Identifier ] [ TestBaudRate ] :Type ] ... [ Type ]

描述

reset 命令链接到 tset 命令。如果 tset 命令在作为 reset 命令运行， 它先执行下列操作，然后再完成所有依靠终端的处理：

  • 将伪造和回显模式设置为打开
  • 关闭 cbreak 和“原始”模式
  • 打开换行转换
  • 恢复特殊字符到敏感状态。

任何查找到的 NULL 或 -1 特殊字符复位为其缺省值。 tset 命令的所有标志可以和 reset 命令一起使用。

当程序死掉并使终端处于不合适的状态时 reset 命令最有用。 可能需要序列 <LF>reset<LF>（其中 <LF> 是 Ctrl-J， 换行）使 reset 命令成功运行，因为回车符在该状态可能不工作。频繁地序列 <LF>reset<LF> 将不回显。

标志

- 所依赖的终端的名称输出到标准输出。 这企图让 shell 捕捉并且放到 TERM 环境变量中。
-e C 设置擦除字符为所有终端上由 C 变量指定的字符。 缺省字符是终端上的退格符，通常是 ^ (下加符)。 字符 C 可以直接输入也可以使用 ^ (下加符) 输入。
-I 抑制终端初始字符串的传输。
-i C 与 -e 标志相似，但使用中断字符而不是擦除字符。 C 变量缺省值是 ^C。 ^ 字符也可以用于该选项。
-k C 与 -e 标志相似，除了使用 行删除字符而不是使用擦除字符。C 变量缺省值是 ^X。如果 -k 未指定，则不管删除字符。 ^ 字符也可以用于该项。
-mIdentifierTestbaudRate：Type 指定何种类型的终端（在Type 参数）通常用在 Identifier 参数标识的端口上。 空的标识符匹配所有的标识符。 可以选择在 TestBaudRate 参数中指定波特率。
-n 在带有 Berkeley 4.3 tty 驱动程序的系统上，指定新建 tty 驱动程序模式应该为该终端初始化。对于 CRT，只有波特率是 1200 bps 或更大时 CRTERASE 和 CRTKILL 模式才设置。 请参阅 tty 文件获得更多信息。
-Q 抑制 Erase set to 和 Kill set to 消息的打印。
-s 基于所依赖的终端名打印 csh 命令序列，该命令初始化 TERM 环境变量。

文件

/usr/share/lib/terminfo/?/* 包含终端能力数据库。
