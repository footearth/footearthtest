nfsd 守护程序

用途

服务客户机对文件系统操作的请求。

语法

/usr/sbin/nfsd [  -a |  -p { tcp | udp } ] [  -c max_connections ] [  -w max_write_size ] [  -r max_read_size ] nservers

描述

在服务器上运行 nfsd 守护程序并处理客户机对文件系统操作的请求。

每个守护程序每次处理一个请求。根据您希望服务器处理的负载量，分配最大数目的线程。

用以下的系统资源控制器（ SRC）命令来启动和停止 nfsd 守护程序：

startsrc -s nfsd

stopsrc -s nfsd

使用 chnfs 命令来更改用 SRC 命令启动的守护程序数目。使用 chssys 命令来更改 SRC 受控守护程序的参数。

标志

-a 指定将服务 UDP 和 TCP 传输。该标志仅适用于 AIX 4.2.1 或更新版本。
-c max_connections 指定 NFS 服务器允许的 TCP 连接的最大数目。该标志仅适用于 AIX 4.2.1 或更新版本。
nservers 指定 NFS 服务器能够处理的并发请求的最大数目。该并发由按需要在内核中创建的 nservers 线程数完成。该标志仅适用于 AIX 4.2.1 或更新版本。
-p tcp or -p udp 将 UDP 和 TCP 传输到 NFS 客户机（缺省）。您只能指定 UDP 或 TCP。例如，如果使用 -p tcp，那么 NFS 服务器只接受使用 TCP 协议的 NFS 客户机请求。该标志仅适用于 AIX 4.2.1 或更新版本。
-r max_read_size 为 NFS V3 指定文件读取请求所允许的最大大小。缺省值和允许的最大值是 32K。该标志仅适用于 AIX 4.2.1 或更新版本。
-w max_write_size 为 NFS V3 指定文件写请求所允许的最大大小。缺省值和允许的最大值是 32K。该标志仅适用于 AIX 4.2.1 或更新版本。

参数

能被更改的参数：

NumberOfNfsds 指定将要启动的守护程序数目。该参数不适用于 AIX 4.2.1 或更新版本。

示例

  1. 要使用 src 命令启动 nfsd 守护程序，请输入： 

    startsrc -s nfsd

    该示例中，startsrc -s nfsd 项启动了守护程序，启动的守护程序的数目是在脚本中指定的。

  2. 要更改在您系统上运行的守护程序数目，请输入： 

    chssys -s nfsd -a 6

    该示例中，chssys 命令将在您系统上运行的 nfsd 守护程序数目更改为 6。
