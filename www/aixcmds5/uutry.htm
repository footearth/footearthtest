uutry 命令

用途

访问一个指定的远程系统，调试打开，允许用户重设缺省重试时间。

语法

uutry [  -xDebugLevel ] [  -r ] SystemName

描述

uutry 命令访问远程系统，远程系统由 SystemName 参数给出且使用调试方式。调试方式提供了一种监控“基本联网实用程序”（BNU）和远程计算机连接以及文件传输的方法。uutry 命令调用 uucico 守护进程来访问远程系统。

调试输出滚动显示在本地系统屏幕上。一旦系统显示信息完毕，按“中断”键返回提示。

如果第一次访问远程系统尝试失败，-r 标志可以重设缺省的重试时间。缺省重试时间是 5 分钟。

SystemName 参数是必需的，用来标明您想访问的远程系统的名称。

注意：
  1. 您必须在 /usr/sbin/uucp 目录中发出 uutry 命令或者您在发出命令时使用全路径 /usr/sbin/uucp/uutry。
  2. uutry 命令是存储在 /usr/lib/uucp 目录中的 shell 脚本。
  3. 如果调试输出滚动速度太快而导致不能阅读，使用 Uutry 命令将输出保存到一个临时文件中。

标志

-r 覆盖缺省重试时间。如果由于某些原因，uucico 守护进程不能完成所要求的连接，则守护进程等候预定的时间之后会重试。缺省重试时间是 5 分钟。

注意：远程系统最近被轮询的时间在 SystemName 文件中记录，此文件在目录/var/spool/uucp/.Status 中。
-xDebugLevel 重设由 uutry 命令显示在屏幕上的调试信息的详细资料数量。DebugLevel 变量的有效范围是 0 到 9，缺省值是 5。数字越大，最后的报告就会越详细。如果 -x 标志没有使用，uucico 守护进程就以缺省级别启动，产生中等的信息量。

示例

  1. 改变详细信息的数量，这些信息是由 uutry 命令提供的关于 uucico 的操作进度，使用 -x 标志指定不同的调试级别。例如，输入：
    /usr/sbin/uucp/uutry  -x9 venus

    指示 uutry 命令生成尽可能多的关于 uucico 守护进程的工作情况信息。

  2. 当第一次访问远程系统失败后，重试访问的缺省时间是 5 分钟。缩短访问远程系统的缺省重试时间，输入：

    /usr/sbin/uucp/uutry  -r venus

    使用 -r 标志指示 uucico 守护进程去访问远程系统venus，重设缺省重试时间。守护进程尝试访问系统 venus，周期性的重试直到连接成功，然后在本地系统的屏幕上显示调试输出。

文件

/usr/sbin/uucp/uutry 包含 uutry 命令。
/etc/uucp/Devices 包含可用设备的信息。
/etc/uucp/Dialcodes 包含拨号代码缩写。
/etc/uucp/Dialers 标明连接中的首次握手。
/etc/uucp/Permissions 描述远程系统的访问权限。
/etc/uucp/Systems 描述可访问的远程系统。
/var/spool/uucp/.Status/SystemName 列出最后一次由SystemName 文件命名的远程系统被访问的时间。
/var/spool/uucppublic/* 包含 BNU 公共目录。
