startrpdomain 命令

用途

使已定义的对等域联机。

语法

startrpdomain [ -A | -L ] [-t timeout] [-h] [-TV] peer_domain

描述

startrpdomain 命令通过启动属于此对等域的每个节点上的资源使已联机定义的对等域联机。

startrpdomain 命令必须在定义到对等域的节点上运行。每次在对等域中运行此命令时，它都请求定义到对等域中的所有脱机节点在对等域中联机。可以在对等域中多次运行此命令。如果在对等域中定义的所有节点已经联机，则不执行操作。

startrpdomain 命令通过在定义到对等域的节点上检测对等域配置从而确定对等域配置以用来使对等域联机。找到最新版本的对等域配置信息用来使对等域联机。在缺省情况下，使用在至少一半的节点上找到的最新版本的对等域配置。指定 -A 标志导致使用在对等域中定义的所有节点上找到的最新版本的对等域配置。指定 -L 标志导致使用本地节点上的配置。

确定最新版本对等域配置信息时，配置超时定义何时停止检查版本并开始将对等域联机。缺省超时值是 120 秒。超时值可用 -t 标志更改。超时值应该至少足以能够从至少一半的节点上找到最新版本的对等域配置信息。

一个节点每次只能在一个对等域上联机。当某节点已与一个对等域联机时，startrpdomain 命令不能运行在该节点上用于另一个对等域。

标志

-A
从对等域的所有节点中查找并使用最新版对等域配置信息。如果指定了 -L 标志，则不能指定该标志。如果没有指定任何一个标志（-A 或 -L），则使用来自对等域中至少一半节点的最新版本对等域配置信息。
-L
使用在本地节点上的最新版本的对等域配置信息。如果指定了 -A 标志，则不能指定该标志。如果没有指定任何一个标志（-A 或 -L），则使用来自对等域中至少一半节点的最新版本对等域配置信息。
-t timeout
指定超时值（秒）。此 标志限制用于查找最新版本的对等域配置的时间。当超过超时值时，使用此时找到的最新版本的对等域配置信息。超时值应该足够长，以便能够从至少一半的节点上找到最新版本的对等域配置信息。缺省超时值是 120 秒。
-h
将命令的用法语句写到标准输出。
-T
将命令的跟踪消息写到标准错误。只限于软件服务组织使用。
-V
将命令的详细消息写到标准输出。

参数

peer_domain
指定要联机的预定义对等域的名称。

安全性

startrpdomain 命令的用户需要对定义到对等域的每个节点上的 IBM.PeerDomain 资源类的具有写权限。缺省情况下，对等域中任何节点上的 root 用户通过配置资源管理器拥有对此资源类的读和写访问权。

退出状态

0
命令成功运行。
1
RMC 出错。
2
命令行界面脚本出错。
3
命令行输入错误标志。
4
命令行输入错误参数。
5
由于错误的命令行输入而出错。
6
对等域定义不存在。

环境变量

CT_CONTACT
确定与资源监视和控制（RMC）守护进程发生会话的系统。当 CT_CONTACT 设置为主机名或 IP 地址时，该命令会与指定主机上的 RMC 守护进程联系。如果未设置 CT_CONTACT，则该命令会与运行它的本地系统的 RMC 守护进程联系。RMC 守护进程会话的目标和管理作用域确定了处理的资源类或资源。

限制

此命令必须从定义到对等域的节点上运行。

标准输入

指定 -f "-" 标志 时，此命令从标准输入读取一个或多个节点名。

标准输出

指定 -h 标志时，此命令的用法语句写到标准输出。所有的详细消息写到标准输出。

标准错误

所有的跟踪消息写到标准错误。

示例

在这些示例中，nodeA 是在 ApplDomain 上定义的节点之一。

  1. 为使 ApplDomain 联机，在 nodeA 上运行此命令： 
    startrpdomain ApplDomain
  2. 要使用对等域中的所有节点以实现 ApplDomain 联机来获取最新版本的对等域配置信息，请在 nodeA 上运行该命令： 
    startrpdomain -A ApplDomain
  3. 要使用 240 秒的对等域配置信息超时值以使 ApplDomain 联机（确保使用对等域中至少一半的节点），请在 nodeA 上运行该命令： 
    startrpdomain -t 240 ApplDomain

位置

/usr/sbin/rsct/bin/startrpdomain
包含 startrpdomain 命令

文件

修改 /etc/services 文件。
