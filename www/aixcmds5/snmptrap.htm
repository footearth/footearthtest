snmptrap 命令

用途

生成一个通知（陷阱），用指定的消息向 SNMP 管理器报告事件。

语法

snmptrap [ -a Host ] [ -c Community ] [-d ] -m Message

描述

生成一个通知（陷阱），用指定的消息向 SNMP 管理器报告事件。

标志

-a Host 指定连接至指定的主机上的 SNMP 代理程序。如果不指定 -a 标志，缺省主机为本地主机。Host 可以是一个 IP 地址或一个主机名。
-c Community 指定要使用的团体名。必须在 SNMP V3 的 /etc/snmpdv3.conf 或 SNMP V1 的 /etc/snmpd.conf 中设置该团体，并且该团体至少具有对运行在指定主机或本地主机上的 SNMP 代理的读访问权限。如果不指定 -c 标志，缺省团体名是“public”。
-d 启用调试工具
-m Message 定义 snmptrap 命令将发送的消息。Message 指定陷阱将保留的信息。该信息是文本格式。-m 标志必须是最后指定的标志。

退出状态

0
陷阱信息正确发出。
1
这表明处理中发生问题。

示例

  1. 为了向本地主机上运行的 SNMP 代理发送带有“hello world”的陷阱，请输入以下命令： 
    snmptrap -m hello world
    注:
    团体“public”必须具有本地主机上运行的 SNMP 代理程序的读访问权限。有关详细信息，请参考 SNMP 配置文档。
  2. 为了向本地主机上运行的 SNMP 代理发送团体名为“community1”并带有“hello world”消息的陷阱，请输入以下命令： 
    snmptrap -c community1 -h blah -m hello world
    注:
    团体“community1”必须具有主机“blah”上运行的 SNMP 代理的读访问权限。详细信息，请参考 SNMP 配置文档。

文件

/etc/snmpdv3.conf 包含 SNMP V3 代理程序的配置文件。
/etc/snmpd.conf 包含 SNMP V1 代理程序的配置文件。
