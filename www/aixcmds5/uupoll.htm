uupoll 命令

用途

强制对远程 BNU 系统轮询。

语法

uupoll [  -gGrade ] [  -n ] SystemName

描述

uupoll 命令使“基本联网实用程序”（BNU）对由 SystemName 参数指定的远程系统进行轮询。此命令通常由 cron 守护进程运行或由想立即执行作业的用户运行。否则，只在由/etc/uucp/Poll 和 /var/spool/cron/crontabs/uucp 文件中调度的时间，uudemon.poll 命令对远程系统进行轮询。

正常的讲，只在由 Poll 文件指定的时间或此远程系统有排队的作业时，uucico 守护进程才访问远程系统。uupoll 命令对远程系统的空作业进行排队然后调用 uucico 守护进程。这就使 uucico 守护进程立刻访问远程系统并尝试对它发送所等候的任何作业。使用 -g 标志指定只发送高优先级作业。

使用 -n 标志对空作业进行排队，而不用启动 uucico 守护进程。选项使用于：

  • 在调用 uucico 守护进程调试以前，对空作业排队。
  • 通常在调用 uucico 守护进程以前对空作业进行排队，然后使守护进程对指定的系统进行轮询。

SystemName 参数是必需的，用于指定要轮询的远程系统的名称。

标志

-gGrade 指示 uupoll 命令只发送已给定优先级的作业（由Grade参数指定）或在此调用中优先级更高的作业。低优先级的作业将保留在队列中直到下一次远程系统再次被轮询。
-n 对空作业排队，但不调用 uucico 守护进程。

示例

  1. 使用 cron 守护进程运行 uupoll 命令，在您的 crontabs 文件中如下放入条目： 

    0  1,7,16  *  *  *  /usr/bin/uupoll hera

    这在以下时间轮询系统 hera：0100 小时（1 a.m.）、0700 小时（7 a.m.）和 1600 小时（4 p.m.）

  2. 如果当地的系统已经在特定的时刻运行 uucico 守护进程，通常您想在现运行的 uucico 守护进程前排队空作业。例如，如果系统以钟点计算运行 uucico 守护进程，在您的 crontabs 文件中如下放入条目：
    0  1,7,16  *  *  *  /usr/bin/uupoll  -n zeus
    0  5,12,21 *  *  *  /usr/bin/uupoll -n hera
    5  *       *  *  *  /usr/sbin/uucp/uucico -r1
    为远程站点在给定时刻进行空作业排队，空作业在给定时刻 5 分钟后由 uucico 守护进程运行处理。
  3. 使 uucico 守护进程传递 N 优先级或更高优先级的所有作业给系统 zeus：
    uupoll  -gN zeus

文件

/usr/bin/uupoll 包含 uupoll 命令。
/etc/uucp/Poll 指定什么时刻 BNU 程序应该轮询远程系统以执行任务。
/var/spool/cron/crontabs/uucp 调度对远程系统的自动轮询。
/var/spool/uucp/SystemName 包含要传递给远程系统的文件。
