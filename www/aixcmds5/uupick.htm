uupick 命令

用途

完成传输并处理 uuto 命令所发出的文件。

语法

uupick [ -sSystem ]

描述

uupick 命令是一种“基本联网实用程序”（BNU）命令，完成传输并处理 BNU uuto 命令发给一个指定用户标识的文件。

一旦该复制文件到达 receive 目录，rmail 命令通知接收人文件已经到达。接收人接着发出 uupick 命令，搜索本地系统里的公共目录以查找以下列名称的形式发送的文件：

/var/spool/uucppublic/receive/User/System/File

对于每一个搜索到的文件或目录，uupick 命令在本地系统的屏幕上显示如下消息：

from System: [file File] [dir Directory]
?

跟在该消息后面的问号提示符（?）暗示你现在可以输入文件处理选项中的任意一项。

标志

-s System 在 /var/spool/uucppublic/receive/User/System 中搜索从指定系统发来的文件。系统名称只包含 ASCII 字符。

文件处理选项

跟在该消息后面的问号提示符（?）指示应输入下列文件处理选项之一：

选项 操作
!Command 退出到 shell 以运行指定的命令。执行完该命令后，用户自动返回到 uupick 命令。
* 显示所有文件处理选项。
a [Directory] 将当前在 receive 目录下的 uuto 文件迁移到本地系统中的一个指定的目录里。缺省情况是当前的工作目录。使用完整或相对路径名来指定目的地目录。
Ctrl-D 停止处理，从 uupick 中退出。
d 删除指定文件。
m [Directory] 将文件迁移到一个指定的目录。如果 Directory 变量没有指定完整的路径名，则指定一个相对于当前目录的目的地址。如果没有给出目的地，则缺省认定为本地系统的当前的工作目录。
new-line 按下 Enter 键则移动到 receive 目录下的下一个条目。
p 在工作站屏幕上显示文件的内容。
q 停止处理，从 uupick 命令中退出。

示例

  1. 要接收一个用 uuto 命令发出的文件并将其添加到当前的工作目录中，输入： 

    uupick

    系统响应的消息类似如下内容：

    from system anchor: file file1
?

    输入：

    a

    在本例中，用 uuto 命令从系统 anchor 发出的文件 /usr/bin/file1 被添加到当前的工作目录中。

  2. 要接收一个用 uuto 命令发出的文件并将其添加到本地系统中一个指定的目录中，输入： 

    uupick

    系统响应的消息类似如下内容：

    from system anchor: file file2
?

    输入：

    a /usr/bin1

    在本例中，用 uuto 命令从系统 anchor 发出的文件 /usr/bin/file2 被添加到本地系统中的 /usr/bin1 目录中。

    注意：a /usr/bin1 指令意味着要移动所有的文件而不只是一个文件。这样，如果 ~/anchor/... 目录中有任何其它文件，则这些文件也要被移动。
  3. 要搜索从系统 anchor 发出的文件，输入：
    uupick  -s anchor
    系统响应的消息类似如下内容： 

    from system anchor: file file1

文件

/usr/bin/uupick 包含 uupick 命令。
/var/spool/uucppublic 包含 BNU 公共目录。
