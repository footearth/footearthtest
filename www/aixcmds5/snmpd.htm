snmpd 守护进程

用途

作为后台进程启动简单网络管理协议（SNMP）代理程序。

语法

参照 snmpdv1 或 snmpdv3 守护进程的语法。

描述

/usr/sbin/snmpd 是符号链接，可链接到仅支持 SNMP V1 的 snmpdv1 守护进程，也可链接到支持 SNMP V3 的加密或不加密的 snmpdv3 守护进程版本。关于 snmpd 守护进程的信息，请参考 SNMP 代理相应版本的文档。在 AIX 5.2 上，系统引导期间运行的缺省 SNMP 代理程序是 SNMP V3 代理程序的非加密版本。

注:
SNMP V3 代理程序的加密版可从“AIX 扩展包”中获取。

文件

/usr/sbin/snmpd 包含可链接到 /usr/sbin/snmpdv1、/usr/sbin/snmpdv3e 或 /usr/sbin/snmpdv3ne 的符号链接。
/usr/sbin/snmpdv1 包含 SNMP V1 代理程序。
/usr/sbin/snmpdv3e 包含 SNMP V3 代理程序的加密版。
/usr/sbin/snmpdv3ne 包含 SNMP V3 代理程序的非加密版。

相关信息

snmpv3_ssw 命令。
