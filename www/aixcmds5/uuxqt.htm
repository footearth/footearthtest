uuxqt 守护进程

用途

执行“基本联网实用程序”（BNU）远端命令请求。

语法

uuxqt [  -e ] [  -sSystemName ] [  -xDebugLevel ]

描述

“基本联网实用程序”（BNU）uuxqt 守护进程在指定的远程系统上执行命令。

每一个联网的系统上的 uuxqt 守护进程定期在假脱机目录中搜索远程执行（X.*）文件。这些文件由uucico 守护进程发送到该目录以响应 uux 命令。

当它查找X.* 文件时，uuxqt 守护进程检查每个文件以确保：

  • 所有要求的数据文件（D.*）都是可用的。
  • 发出要求的系统有必要的权限以访问该数据文件和运行所请求的命令。

    注：uuxqt 守护进程使用 /etc/uucp/Permissions 文件来验证文件的可访问性和命令的执行权限。

如果数据文件存在，而且发出请求的系统有相应的权限，则 uuxqt 守护进程执行命令。

注：uuxqt 命令通常从一个 shell 过程，uudemon.hour 命令执行，而不是从命令行输入。您必须具有 root 用户特权以从命令行发出 uuxqt命令。

标志

-e 启用文件扩展。
-sSystemName 指定要联系的远程系统。只有在手动启动 uuxqt 命令时才使用。系统名称在 uuxqt 命令自动启动时是内部提供的。

注：系统名称只能包含 ASCII 字符。
-xDebugLevel 在本地系统的屏幕上显示调试信息。DebugLevel 变量值是一个从 0 到 9 的整数，缺省值是 5。DebugLevel 变量的值越大，调试信息就越详细。

安全性

访问控制：您必须具有 root 用户权限以从命令行启动 uuxqt 守护进程。

示例

要启动 uuxqt 守护进程进行调试，输入：

/usr/sbin/uucp/uuxqt  -svenus  -x7

这指示该命令联系远程系统 venus 并提供有关此次联系的相当详细的信息。

文件

/usr/sbin/uucp/uuxqt 包含 uuxqt 守护进程
/etc/locks 包含防止多重使用设备和多重调用系统的锁定文件。
/etc/uucp/Maxuuxqts 限制远程命令的执行。
/etc/uucp/Permissions 描述远程系统的访问权限。
/var/spool/uucp/* 包含执行文件和数据文件。
