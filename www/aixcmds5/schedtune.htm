schedtune 命令

用途

设置 CPU 调度程序和虚拟内存管理器参数的兼容性脚本。

语法

schedtune [ -D ]

schedtune [ -d Number ] [ -e Number ] [ -f Number ] [ -h Number ] [ -m Number ] [ -p Number ] [ -r Number ] [ -t Number ] [ -w Number ]

描述

schedtune 样本程序正在逐渐淘汰，将来的发行版将不支持该样本程序。它正在被 schedo 命令所代替，该命令能够用于和先前发行版中的 schedtune 一样设置相同的参数。在 AIX 5.2 中，提供了调用 schedo 的兼容性脚本来帮助转换。

只有 root 用户才能执行 schedtune 命令。schedtune 命令所进行的更改会持续到下一次重新引导系统。在 AIX 5.2 中，schedo 可以用于进行永久更改。

注意: 错误使用该脚本可能导致性能下降或操作系统故障。在使用 schedtune 更改系统参数之前，确保您您已经学习了《AIX 5L V5.2 性能管理指南》中相应的调整部分。

标志

如果没有指定标志，则调用 schedo -a 以显示当前值。

-D 调用 schedo -D 来恢复缺省值。
-a Number 调用 schedo -o affinity_lim=Number 来设置上下文转接编号，其后 SCHED_FIF02 策略不再支持线程。
-b Number 调用 schedo -o idle_migration_barrier=Number 来设置空闲迁移障碍。
-c Number 调用 schedo -o %usDelta=Number 来控制时钟漂移的调整。
-d Number 调用 schedo -o sched_D=Number 来设置用于衰减 CPU 使用的因子。
-e Number 调用 schedo -o v_exempt_seconds=Number 来设置最近暂挂和恢复的进程可被选择重新暂挂之前的时间。
-f Number 调用 schedo -o pacefork=Number 设置重新创建失败的子进程前时钟信号延迟数。
-F Number 调用 schedo -o fixed_pri_global=Number 将固定优先级线程保持在全局运行队列中。
-h Number 调用 schedo -o v_repage_hi=Number 来更改用于确定进程暂挂何时开始和结束的系统范围准则。
-m Number 调用 schedo -o v_min_process=Number 来设置最小的多道程序设计级别。
-p Number 调用 schedo -o v_repage_proc=Number 来更改用于确定暂挂哪些进程的每个进程的标准。
-r Number 调用 schedo -o sched_R=Number 来设置累加 CPU 使用的速率。
-s Number 调用 schedo -o maxspin=Number 来设置休眠前在某个锁上轮转的次数。
-t Number 调用 schedo -o timeslice=Number 来设置 10 毫秒的时间片数。
-w Number 调用 schedo -o v_sec_wait=Number 设置在系统颠簸结束之后将进程加回到 mix 之前所等待的秒数。
-? 显示该命令及其参数的简要描述。

请参阅 schedo 命令的可调参数部分。
