statd 守护进程

用途

为 NFS 上的锁定服务提供崩溃和恢复功能。

语法

/usr/sbin/rpc.statd

描述

statd 守护进程与 lockd 守护进程相互作用，为网络文件系统（Network File System, NFS）上的锁定服务提供崩溃和恢复功能。statd 守护进程应总是在 lockd 守护进程之前启动。

statd 守护进程由下列 SRC 命令启动和停止：

startsrc -s rpc.statd

stopsrc -s rpc.statd

状态监视器保持有关连接位置的信息以及 /var/statmon/sm 目录、/var/statmon/sm.bak 目录和 /var/statmon/state 文件中的状态。当重新启动时，statd 守护进程查询这些文件，并尝试重建它在终止前所拥有的连接。要在事先不了解现有的锁或状态的情况下重新启动 statd 守护进程随后启动 lockd 守护进程，请在重新启动 statd 守护进程前将这些文件删除。
